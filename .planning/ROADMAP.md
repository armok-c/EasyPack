## v2.1 Phase Details

### Phase 21: 打包版本号更新

**Goal:** 构建时自动更新 tauri.conf.json 中的版本号

**Requirements:** VER-05

**Success criteria:**
1. 运行 `npm run tauri build` 时自动更新 `src-tauri/tauri.conf.json` 的 `version` 字段
2. 版本号来源与 package.json 的 version 保持同步
3. 构建产物（安装包）版本信息与实际版本一致

**Plans:** 1/1 plans complete

Plans:
- [x] 21-01-PLAN.md — 创建版本同步脚本并接入 Tauri 构建流程

---

### Phase 22: 全局指令移除与重构

**Goal:** 移除全局指令系统，将项目指令栏目迁移至全局指令原位置并改名为"项目环境"

**Requirements:** CMD-09, CMD-10

**Success criteria:**
1. MainArea 中全局/项目切换按钮组已完全移除
2. `commandMode` 状态和所有全局指令 CRUD 逻辑已清理
3. `CommandItem.scope` 类型不再包含 `"global"` 值
4. 原"项目指令"栏目移至全局指令原位置，UI 标签显示为"项目环境"
5. 现有项目级指令数据无损迁移，功能正常

**Plans:** 3/3 plans complete

Plans:
**Wave 1**
- [x] 22-01-PLAN.md — 数据类型更新 + 启动时旧全局指令清理

**Wave 2** *(blocked on Wave 1 completion)*

### Phase 23: 环境标签页与管理

**Goal:** 实现环境标签页切换、管理环境模态窗、环境切换下拉栏。环境用于管理项目的配置文件（如 `.env`），每个环境保存一组文件的副本，切换环境可将对应文件内容写入项目根目录。

**Requirements:** ENV-01, ENV-02, ENV-08

**Success criteria:**
1. 项目环境内页顶部横向标签页可左右滚动切换浏览不同环境的文件
2. 点击标签页切换当前浏览的环境，下方文件列表随之更新（指令卡片不变）
3. "管理环境"模态窗支持新增环境（名称输入），列表显示名称/创建时间/修改时间/操作
4. 重命名环境功能正常，删除环境有二次确认弹窗
5. 环境切换栏：下拉框选择目标环境 + 启用按钮，点击启用将所选环境的配置文件内容写入项目根目录

**Plans:** 3/3 plans complete

Plans:
**Wave 1**
- [x] 23-01-PLAN.md — 数据类型（Environment/ManagedFile）+ Rust 文件读写命令（read_file_content/write_file_content）
- [x] 23-02-PLAN.md — useProject 环境状态管理（CRUD + apply + 持久化 + 生命周期集成）

**Wave 2** *(blocked on Wave 1 completion)*
- [x] 23-03-PLAN.md — UI 组件（EnvTabBar/EnvSwitchBar/ManageEnvDialog）+ MainArea/App.tsx 集成

---

### Phase 24: 文件管理与编辑器

**Goal:** 文件添加/删除/列表查看/语法高亮编辑功能

**Requirements:** ENV-03, ENV-04, ENV-06, ENV-07

**Success criteria:**
1. "添加"按钮可向当前环境添加需要管理的配置文件
2. "删除"按钮支持列表多选文件，二次确认弹窗后删除
3. 文件列表正确显示勾选框、文件名+后缀、修改时间、查看按钮
4. 查看按钮打开文本模态窗，支持 xml/yml/json/toml 语法高亮（CodeMirror 6）
5. 编辑器支持语法错误检测，错误行有视觉标记
6. 右下角保存按钮将修改保存到当前环境存储的副本（不直接写磁盘，需通过 ENV-08「启用」才应用到项目目录）

---

### Phase 25: 同步差异对比

**Goal:** 跨环境配置文件差异对比与同步

**Requirements:** ENV-05

**Success criteria:**
1. 勾选文件后"同步差异"按钮可点击，未勾选时禁用
2. 点击按钮弹出环境选择弹窗，显示全部其他环境（可多选）
3. 选择目标环境后展示 IDE Git 冲突风格对比模态窗（@git-diff-view/react）
4. 对比视图支持 split/unified 双模式，内置语法高亮
5. 差异对比内容可复制/查看，支持暗色主题
