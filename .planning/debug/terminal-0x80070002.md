## Resolution

root_cause: |
  wt.exe (位于 WindowsApps 目录的 UWP App Execution Alias) 从 Tauri GUI 子系统进程中无法正确
  传递和执行命令参数。spawn() 调用本身成功（进程创建成功），但 WT 内部无法正确解析和执行传入
  的命令行。这是 WT/UWP App Execution Alias 在非交互式上下文中的已知限制。
  前四次修复尝试都在错误的方向上（修改参数格式），真正的问题是 wt.exe 本身在这种场景下不可用。

fix: |
  移除 WT 优先路径，仅使用 cmd.exe /C start 方式启动命令。
  cmd /C start "" /d path cmd /K command — 通过 start 在新窗口中启动 cmd，
  使用 /d 指定工作目录，/K 保持窗口打开。
