### 修改 src/App.tsx

   ```typescript

   ```

2. **调用 useFloatWindow hook**：在 useTray 调用之前（hook 调用顺序不影响逻辑，但建议与其他窗口管理 hook 放在一起）：
   ```typescript
   // Phase 13: 悬浮窗
   const { floatVisible, toggleFloat, destroyFloat } = useFloatWindow({
     currentProject,
     commands,
     onExecute: handleExecuteWithRecent,
     appWindow,
   });
   ```

3. **修改 TitleBar 组件调用**：传递新增的 props：
   ```tsx
   <TitleBar
     onSettingsOpen={() => setSettingsOpen(true)}
     onFloatToggle={toggleFloat}
     floatVisible={floatVisible}
   />
   ```

4. **修改 useTray 调用**：添加新增的参数：
   ```typescript
   useTray({
     currentProject,
     commands,
     recentCommands,
     visibility,
     onExecute: handleExecuteWithRecent,
     onShow: () => { showFromTray(); appWindow.show().catch(console.error); appWindow.setFocus().catch(console.error); },
     onHide: () => { hideToTray(); appWindow.hide().catch(console.error); },
     onQuit: async () => {
       await destroyFloat();  // D-12: 主窗口退出时销毁悬浮窗
       await appWindow.destroy();
     },
     enabled: trayEnabled,
     appWindow,
     onToggleFloat: toggleFloat,
     floatVisible,
   });
   ```

   注意：`onQuit` 回调中先调用 `destroyFloat()` 再 `appWindow.destroy()`，确保悬浮窗在主窗口退出前被清理（D-12：主窗口退出时悬浮窗一起关闭）。

5. **不需要修改 onCloseRequested**：当前的 onCloseRequested 处理器在 closeToTray=true 时调用 `event.preventDefault()` + `appWindow.hide()`，这只是隐藏主窗口到托盘，不会退出应用。悬浮窗独立存活（D-12：主窗口关闭到托盘时悬浮窗独立存活）。当 closeToTray=false 时，不拦截关闭请求，走正常关闭流程，最终触发 onQuit 中的 destroyFloat。

6. **验证 hook 调用顺序不影响渲染**：useFloatWindow 使用 useCallback 和 ref，不依赖其他 hook 的返回值。放在 useTray 之前或之后都可以正常工作。
