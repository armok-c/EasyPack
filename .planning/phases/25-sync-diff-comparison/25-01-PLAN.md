    - 由于 @git-diff-view/react 不直接提供 hunk 级别的自定义按钮插槽，采用替代方案：

      - 每个 hunk 的操作条显示「← 使用源」和「使用目标 →」按钮

    - 「← 使用源」：
      - 从源文件内容提取此 hunk 对应的文本，替换目标文件对应区域

      - 创建 undo snapshot

      - 标记为已解决（不修改内容）
      - toast：「已确认目标内容」
    - 「撤销」：
      - 从 undoSnapshots 恢复 target 内容
      - 调用 onUpdateFile
      - 标记为未解决
      - toast：「已撤销变更」

    **6. 底部状态栏（D-27）：**
    - height: h-9, border-t border-border

    打开时（open=true）：
    1. 对每个 (fileName, envId) 对，调用 prepareDiffData 预处理

    3. 目标内容：targetEnv.files.find(f.name === fileName)?.content ?? ""
    4. 缓存到 diffDataCache
    5. 默认选中第一个文件 + 第一个目标环境

    关闭时（open=false）：
    1. 静默关闭（D-26），已应用变更保留
    2. 不提示未解决变更

    注意：由于 @git-diff-view/react 的具体 API 可能在 v0.1.5 版本有差异，代码需灵活适配。
    DiffFile 上可能没有 buildSplitDiffLines 等方法 — 运行时验证。准备好使用组件最简配置。
