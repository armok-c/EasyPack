## Detailed Component Contracts

### 1. SyncDiffButton (in FileList toolbar)

**Location:** FileList toolbar, between file count span and "添加文件" button.

```
[3 个文件] [ 同步差异 ] [ 添加文件 ] [ 删除 ]
```

**States:**
- **Enabled:** `variant="default"` (primary white), `size="sm"`, `GitCompare` icon on left. At least one file checked.
- **Disabled:** Same layout but `disabled={true}`. Grayed with `opacity-50 cursor-not-allowed`. Tooltip: "请先勾选要对比的文件".
- **Edge case — no other environments exist:** Button still enabled (EnvSelectDialog will show empty list with disabled confirm button).

**Props to add to FileListProps:**
```typescript
onSyncDiff: (checkedFiles: string[]) => void;
```

### 2. EnvSelectDialog (Environment Selection)

**Trigger:** Click on SyncDiffButton opens this dialog.

**Layout:**
- Width: `sm:max-w-[480px]` (medium fixed width per CONTEXT.md D-04)
- Title: "同步差异 — {sourceEnvName}" (source env = current active browsing env)
- Content: Multi-select list with select-all header

**List rows** (CONTEXT.md D-01):
```
☐ 全选                                         (header row)
☐ staging · 3 个匹配, 2 个缺失                 (env item)
☐ production · 5 个匹配, 0 个缺失              (env item)
☐ dev · 0 个匹配, 3 个缺失                     (env item with FileX icon)
```

**Each row:**
- Checkbox (shadcn Checkbox or native input like FileList uses)
- Environment name (truncated with ellipsis, max one line, hover tooltip shows full name per D-07)
- Dot separator
- Matching file count: `{N} 个匹配`
- Missing file count: `{N} 个缺失` (shows FileX icon + count if > 0)
- List sorted alphabetically per D-07

**Header row:**
- Select-all checkbox (when checked, selects all envs; when some selected, indeterminate state)
- `text-xs text-muted-foreground` label

**Footer:** Two buttons — "取消" (outline) | "确认" (primary, disabled when no env selected, tooltip "请至少选择一个目标环境")

**Behavior:**
- Opens empty — all checkboxes unchecked per D-08
- Source environment is excluded from list (can't diff with self)
- Selecting an env with zero matching files is allowed (will show missing file view)
- At least 1 env must be checked for "确认" to enable

### 3. DiffViewDialog (Main Comparison Modal)

**Layout:** Matches FileEditorDialog pattern (CONTEXT.md D-17).
```
┌─────────────────────────────────────────────┐
│ ✕  差异对比 — .env (development → staging)  │ <- DialogHeader + DialogTitle (dynamic)
│    development → staging                     │ <- subtitle (text-xs text-muted-foreground)
├─────────────────────────────────────────────┤
│ [ .env  ] [ config.yml ] [ .env.production ] │ <- File tabs (shadcn TabsList)
├─────────────────────────────────────────────┤
│ [ staging ] [ production ]                   │ <- Env sub-tabs (inside file tab content)
├─────────────────────────────────────────────┤
│                                             │
│  ← 使用源  │  Diff View Content   │ 使用目标 →│ <- @git-diff-view/react + hunk buttons
│            │  (split view)        │           │
│            │                      │           │
├─────────────────────────────────────────────┤
│ 已解决: 2 / 未解决: 5          全部已解决    │ <- Status bar
└─────────────────────────────────────────────┘
```

**Dimensions:** `sm:max-w-[80vw] max-h-[80vh] h-[80vh] flex flex-col` (same as FileEditorDialog).

#### 3a. File Tabs (Top Level)

- Uses shadcn/ui `Tabs` component
- Tab order: user's check order in FileList (CONTEXT.md D-15)
- Each tab label: file basename (last path segment)
- Overflow: `overflow-x: auto` with hidden scrollbar per D-15
- No close button on individual tabs (CONTEXT.md D-26 — close modal entirely)
- No unresolved badge on file tabs per D-16
- Missing files shown with `FileX` icon next to name per D-36
- Only 1 file: still renders the tab bar with a single tab per D-14

#### 3b. Environment Sub-Tabs (Per File)

- Each file tab contains its own set of environment sub-tabs
- Uses nested shadcn/ui `Tabs`
- Each sub-tab = one selected target environment
- No progress badges per D-16
- When resolved count changes for a file/env pair, only that pair's state updates (D-28)

#### 3c. Diff View Content Area

- Uses `@git-diff-view/react` `<DiffView>` component
- Default mode: **Split** (side-by-side) — more natural for config comparison
- Toggle button in the title bar area to switch between Split/Unified
- Theme: `diffViewTheme="dark"` (dark from shadcn zinc preset)
- Syntax highlighting: `diffViewHighlight={true}`
- Font size: `diffViewFontSize={12}` (12px for monospace diff content)
- Line wrapping: `diffViewWrap={false}` (config files are typically short lines)
- Source file shown as `oldFile` (left side in split mode) — read-only reference
- Target file shown as `newFile` (right side in split mode) — modifiable target

**Data flow per CONTEXT.md D-37/D-38:**
1. Source content: `Environment.files.find(f => f.name === fileName).content`
2. Target content: `targetEnv.files.find(f => f.name === fileName).content`
3. Use `generateDiffFile()` from `@git-diff-view/file` to compute diff from two content strings
4. Pass generated `DiffFile` instance to `<DiffView diffFile={...} />`

**Hunk Copy Buttons** (CONTEXT.md D-20):

Each hunk (diff block) gets two action buttons:
```
┌─ Hunk header ──────────────────────────────────┐
│ [← 使用源]  @@ -1,5 +1,6 @@                    │
│ ...                                             │
└─────────────────────────────────────────────────┘
                                               [使用目标 →]
```

- "← 使用源": Replaces target content with source content for this hunk. Visible on all hunks where source != target.
- "使用目标 →": Replaces source content with target content. Actually in D-21, "仅目标环境可变，源环境只读参考", so "使用目标 →" is visually present but conceptually means "reject source change" — it keeps the target as-is (marks hunk resolved without modification).
- Buttons positioned: "← 使用源" on left edge of hunk header, "使用目标 →" on right edge of hunk header (like GitHub merge conflict UI).
- Button style: small outline buttons, `size="sm"` `variant="outline"` reduced to `h-6 text-xs`.

**Resolved/Unresolved States** (CONTEXT.md D-25):
- **Unresolved (default):** Hunk has default `bg-card` background, `border-border` border.
- **Resolved:** Hunk gets `bg-green-500/10` background with `border-green-500/30` left border accent. Green tint indicates the hunk has been addressed.

**Behavior on hunk apply** (CONTEXT.md D-22/D-23):
1. Click "← 使用源" → immediately write source hunk content to target via `onUpdateFile(envId, fileName, newContent)`
2. Click "使用目标 →" → keep target (mark resolved without write)
3. Show toast: "已应用源变更 (可撤销)" or "已确认目标内容"
4. Mark hunk as resolved (green bg)
5. Update status bar counts
6. No secondary confirmation dialog (D-23)

**Undo** (CONTEXT.md D-24):
- Each applied hunk stores previous content snapshot
- After resolution, "← 使用源" / "使用目标 →" buttons are replaced with a single "撤销" button
- Click "撤销" → restore previous target content, revert hunk to unresolved
- Toast: "已撤销变更"

## Interaction Details

### FileList Integration Flow

```
User checks files in FileList
  → SyncDiffButton enables (one or more files checked)
  → User clicks "同步差异"
  → EnvSelectDialog opens (source env excluded from list)
  → User selects target envs, clicks "确认"
  → DiffViewDialog opens with first file + first target env selected
```

### Dialog Navigation Flow

- **File tab click:** Switches to that file's content with its first target env sub-tab active
- **Env sub-tab click:** Switches to that target env's diff for the current file
- **Close (X / Escape):** Closes entire DiffViewDialog per D-26. No "unsaved changes" prompt — all changes are persisted immediately per D-22. Unresolved hunks remain as-is; applied changes stay saved.

### Keyboard Navigation

| Action | Shortcut |
|--------|----------|
| Close dialog | Escape |
| Save/apply current hunk | Not applicable (immediate persistence per D-22) |
| Switch file tab | Arrow keys when focused on tab list |
| Switch env sub-tab | Arrow keys when focused on env tab list |

### `@git-diff-view/react` Configuration

```tsx
import { DiffView, DiffModeEnum } from "@git-diff-view/react";
import { generateDiffFile } from "@git-diff-view/file";
// CSS import: @import "@git-diff-view/react/styles/diff-view.css"

// Pre-compute diff data
const diffFile = generateDiffFile(
  sourceFileName,
  sourceContent,
  targetFileName,
  targetContent,
  fileLang,   // from getLanguageExtension helper
  fileLang    // same lang for both
);
diffFile.initTheme("dark");
diffFile.init();
diffFile.buildSplitDiffLines();   // pre-build for default split view
diffFile.buildUnifiedDiffLines(); // pre-build for toggle

// Render
<DiffView
  diffFile={diffFile}
  diffViewMode={currentMode}            // DiffModeEnum.Split | DiffModeEnum.Unified
  diffViewTheme="dark"
  diffViewHighlight={true}
  diffViewFontSize={12}
  diffViewWrap={false}
/>
```

**View mode toggle:** A small button group in the diff view toolbar area (between file tabs and env sub-tabs, or in dialog title bar):
- "Split" button | "Unified" button
- Active mode highlighted with accent color
- Toggles `DiffModeEnum` value
