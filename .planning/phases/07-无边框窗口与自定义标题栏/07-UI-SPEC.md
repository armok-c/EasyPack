## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- **Title bar height: 28px** (per CONTEXT.md D-01 — compact style to maximize content area). 28px is not a multiple of 4 but is locked by user decision.
- **Window control button hit area: 28px width x 28px height** (matches title bar height, ensures adequate touch/click target).

---

## Typography

Title bar introduces one new typography role. All other typography roles remain unchanged from existing codebase.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Existing — card text, sidebar items |
| Label | 12px | 500 (medium) | 1.2 | Existing — input labels, badges |
| Heading | 16px | 600 (semibold) | 1.3 | Existing — section headers, dialog titles |
| **TitleBar** | **13px** | **500 (medium)** | **1.0** | **NEW — application name in title bar** |

Title bar typography rationale:
- 13px is smaller than body (14px) to reinforce that the title bar is a system-level element, not content.
- Medium weight (500) provides subtle emphasis without competing with headings.
- Line height 1.0 (28px container height / 28px = 1.0 effectively, with vertical centering via flexbox `items-center`).

## Title Bar Component Contract

### Structure (per CONTEXT.md decisions)

```
+---------------------------------------------------------------------+
| [icon] EasyPack                              [─]  [□]  [x]         |  28px height
+---------------------------------------------------------------------+
| Sidebar (240px)  |  Main Content Area                                |
|                  |                                                    |
+---------------------------------------------------------------------+
```

### Component: TitleBar

**File:** `src/components/TitleBar.tsx`

**HTML structure (prescriptive):**
```html
<div class="flex items-center h-[28px] select-none shrink-0" data-tauri-drag-region>
  <!-- Left: App icon + name (draggable via parent) -->
  <div class="flex items-center gap-[6px] pl-[10px]" data-tauri-drag-region>
    <PackageIcon class="w-[14px] h-[14px] text-foreground/80" />
    <span class="text-[13px] font-medium text-foreground">EasyPack</span>
  </div>

  <!-- Spacer: fills remaining width (draggable) -->
  <div class="flex-1" data-tauri-drag-region></div>

  <!-- Right: Window control buttons (NOT draggable) -->
  <div class="flex items-center h-full">
    <button class="titlebar-button" onClick={minimize}>
      <Minus class="w-[12px] h-[12px]" />
    </button>
    <button class="titlebar-button" onClick={maximize}>
      <Square class="w-[12px] h-[12px]" />
    </button>
    <button class="titlebar-button close-button" onClick={close}>
      <X class="w-[12px] h-[12px]" />
    </button>
  </div>
</div>
```

### Window Control Button Styling

Each button:
- Width: 28px (auto from `h-full` + square aspect)
- Height: 28px (inherits from parent `h-full`)
- Icon size: 12px x 12px (proportional to 28px button)
- Default: transparent background, `--color-muted-foreground` icon
- Hover (minimize/maximize): `--color-accent` background, `--color-foreground` icon
- Hover (close): `oklch(0.5 0.15 25)` (red), white icon
- Transition: `background-color 100ms ease`
- Cursor: `default` (not pointer — system buttons use default cursor)

### Drag Region Behavior (per D-05)

- The entire title bar row is draggable via `data-tauri-drag-region` on the container div.
- Window control buttons are NOT draggable — clicking them triggers minimize/maximize/close.
- The left section (icon + name) and spacer are draggable.
- Sidebar and main content area are NOT draggable.

### Double-Click Title Bar

- Double-clicking the drag region should toggle maximize/restore (standard Windows behavior).
- Tauri 2 may handle this automatically via `data-tauri-drag-region`. If not, implement via `onDoubleClick` handler calling the maximize toggle.

## Tauri Configuration Contract

### Window Config Changes (tauri.conf.json)

```json
{
  "app": {
    "windows": [
      {
        "label": "main",
        "title": "EasyPack",
        "width": 720,
        "height": 480,
        "minWidth": 600,
        "minHeight": 400,
        "resizable": true,
        "center": true,
        "decorations": false,
        "shadow": true
      }
    ]
  }
}
```

- `decorations: false` — removes native title bar (WIN-01)
- `shadow: true` — preserves window shadow on Windows 11 DWM (WIN-02, D-07)
- Windows 10 may not render shadow — this is acceptable per D-07

### Window Control Tauri API

Use `@tauri-apps/api/window` for minimize/maximize/close:

```typescript
import { getCurrentWindow } from "@tauri-apps/api/window";

const appWindow = getCurrentWindow();

// Minimize
appWindow.minimize();

// Toggle maximize/restore
appWindow.toggleMaximize();

// Close
appWindow.close();
```

No Rust backend commands needed — these are pure frontend Tauri API calls.

## High DPI Support (WIN-03)

| Aspect | Implementation |
|--------|---------------|
| WebView scaling | Tauri 2 defaults to system DPI scaling. No manual configuration needed. |
| Font rendering | System font stack ("Segoe UI") is DPI-aware on Windows. |
| Icon rendering | lucide-react SVGs scale cleanly at any DPI. |
| Title bar dimensions | 28px height is in CSS pixels; Tauri scales to physical pixels automatically. |
| Border radius | Window rounded corners are handled by Windows 11 DWM automatically. |

Verify on: 1x (96 DPI), 1.5x (144 DPI), 2x (192 DPI) displays.
