    - package.json（version 字段作为数据源）
  </read_first>
  <action>
    在项目根目录 `scripts/` 下创建 `sync-version.js`（若 scripts 目录不存在则创建）。

    **脚本行为（严格按 CONTEXT.md 决策实现）：**

    1. 读取 `process.cwd() + '/package.json'` 解析 JSON，取 `version` 字段作为源版本（D-07：不校验 semver，原样使用）。
    2. 定义目标文件列表：
       - `src-tauri/tauri.conf.json`（JSON，仅替换 `"version": "X.X.X"` 的值部分，用正则匹配 `"version"\s*:\s*"[^"]*"`，替换为 `"version": "${sourceVersion}"`，保留原缩进和所有其他字段）
       - `src-tauri/Cargo.toml`（纯文本，用正则 `/^version\s*=\s*"[^"]*"/m` 匹配替换）
    3. 对每个目标文件：
       a. 读取当前内容
       b. 检查当前版本是否已等于源版本（D-08：相同则跳过此文件，不写入、不输出此文件条目）
       c. 不同时执行替换写入
    4. 写入后不还原（D-02：同步后保留更新版本，下次发布只需改 package.json 一处）

    **错误处理（严格按 D-09 ~ D-11）：**
    - `package.json` 不存在或 JSON 解析失败 → `console.error('Error: Cannot read package.json: ' + detail)` → `process.exit(1)`
    - `tauri.conf.json` 或 `Cargo.toml` 不存在或无法写入 → `console.error('Error: Cannot read/write ' + filepath + ': ' + detail)` → `process.exit(1)`
    - `Cargo.toml` 中 `version` 字段不匹配（未找到） → `console.error('Error: version field not found in Cargo.toml')` → `process.exit(1)`

    **输出格式（D-12）：**
    - 有变更时输出一行到 stdout：`Version synced: ${sourceVersion} -> tauri.conf.json, Cargo.toml`
    - 无需变更时无输出（静默成功）
    - 错误全部输出到 stderr

    脚本必须是零外部依赖（仅使用 Node.js 内置 `fs` 和 `path` 模块）。
    脚本编码：UTF-8，行尾 LF。
  </action>
