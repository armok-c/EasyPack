### Color Marker Application (per D-02)

Implementation: `before:` pseudo-element approach.

```
before:absolute before:left-0 before:top-1 before:bottom-1 before:w-[3px]
before:rounded-l-lg before:transition-colors before:duration-150
```

Color applied via inline style: `style={{ '--tw-before-bg': project.color }}` or direct `before:bg-[${project.color}]`.

The before pseudo-element overlays on top of the existing border, does not modify the border property itself. This avoids Tailwind CSS v4 border-color cascade issues.

## Interaction Specifications

### 1. Project Icon & Color Marker (PROJ-05)

#### Right-Click Menu Flow

```
User right-clicks project row
  -> ContextMenu appears with single item "设置图标和颜色"
  -> User clicks menu item
  -> ProjectSettingsDialog opens
  -> Dialog shows: icon grid (10 icons, 5 cols) + color swatches (8 dots, 4 cols) + preview area
  -> User selects icon and/or color
  -> Preview area updates live (icon + color bar combination)
  -> User clicks "保存设置" -> callback fires with { icon, color }
  -> Toast: "已更新项目样式"
  -> User clicks "取消" or clicks outside -> dialog closes, no change
```

#### ProjectSettingsDialog Layout

```
+------------------------------------------+
|  设置图标和颜色                    [X]    |
+------------------------------------------+
|                                          |
|  图标                                    |
|  [Terminal] [Code] [Server] [Zap] [Git] |
|  [Package] [Globe] [Wrench] [Rocket] [P] |
|                                          |
|  颜色                                    |
|  [红] [橙] [黄] [绿]                    |
|  [蓝] [紫] [粉] [青]                    |
|                                          |
+------------------------------------------+
|  预览                                    |
|  +-----------------------------+         |
|  | [icon] 项目名称             |         |
|  +-----------------------------+         |
+------------------------------------------+
|              [取消]  [保存设置]           |
+------------------------------------------+
```

#### Icon Picker Behavior (matches CommandDialog pattern)

| State | Style |
|-------|-------|
| Default | bg-white/5, rounded-lg |
| Hover | bg-white/10 |
| Selected | bg-white/15, border border-primary, ring-1 ring-primary/50 |
| Focus | focus-visible:ring-2 focus-visible:ring-ring |

#### Color Swatch Behavior

| State | Style |
|-------|-------|
| Default | 24px circle (rounded-full), bg-[color], opacity-70 |
| Hover | opacity-100, scale-110, transition 150ms |
| Selected | opacity-100, ring-2 ring-white/60, scale-110 |
| No color selected | First state: no ring, but a "无颜色" option with dashed border circle |

Note: A "clear color" option (dashed border circle) should be provided to let users remove color marking.

#### Preview Area

Shows a mini sidebar project row with:
- Selected icon on the left of project name
- Selected color as left border (3px bar)
- Project name text (uses current project's name)

### 2. Drag-and-Drop Reorder (PROJ-06)

#### Drag Handle Visual

| State | Style |
|-------|-------|
| Default | opacity-0, absolute positioned left side |
| Group hover | opacity-100, transition-opacity 150ms |
| Hover on handle | text-foreground (brighter), cursor-grab |
| Active drag | cursor-grbing |
| Dragging item | opacity-50 on source, no scale change |

#### Drag Handle Position

The GripVertical icon is positioned to the left of the project name text, mirroring the X delete button on the right side. Layout order within the project row:

```
[GripVertical (left)] [Icon (if set)] [Project Name (flex-1)] [X delete (right)]
```

All three overlays (GripVertical, icon, X) are within the same flex row.

#### Drag Placeholder

During drag:
- Source item: opacity-50
- Drop target: subtle border-dashed border-white/20 indicator (or rely on @dnd-kit built-in visual)

#### Drag Completion (per D-10, D-11)

- On drag end: splice array to new order -> setProjects(newOrder) -> store.set(PROJECTS_KEY, newOrder)
- No intermediate "saving" state — immediate persistence
- Focus moves to the dragged item's new position

### 3. Keyboard Navigation (UI-03)

#### Focus Zone Model (per D-15)

Two distinct focus zones:
1. **Sidebar zone** — projects list + add button
2. **Main area zone** — command cards grid

Tab key cycles between zones (not between every individual element).

#### Sidebar Zone Keys

| Key | Action |
|-----|--------|
| ArrowDown | Move focus to next project (wraps to top) |
| ArrowUp | Move focus to previous project (wraps to bottom) |
| Enter | Select the focused project |
| Tab | Move focus to main area zone |
| Shift+Tab | Move focus to main area zone (reverse) |
| Delete / Backspace | Not handled (delete via X button only) |

#### Main Area Zone Keys

| Key | Action |
|-----|--------|
| ArrowRight | Move focus to next card (+1 index) |
| ArrowLeft | Move focus to previous card (-1 index) |
| ArrowDown | Move focus to card below (+column count) |
| ArrowUp | Move focus to card above (-column count) |
| Enter | Execute focused card's command |
| Tab | Move focus back to sidebar zone |
| Shift+Tab | Move focus back to sidebar zone (reverse) |

#### Global Shortcut Keys (per D-13)

| Key | Action | Guard Conditions |
|-----|--------|-----------------|
| 1-9 | Execute Nth command card directly | Must have: selected project, not in input/textarea, not in edit mode, no dialog open |

Guard conditions (all must be false for shortcut to fire):
- `e.target instanceof HTMLInputElement`
- `e.target instanceof HTMLTextAreaElement`
- `editMode === true`
- `dialogOpen === true` (any dialog including ProjectSettingsDialog)

#### Numeric Shortcut Badge on Cards

Cards show a small numeric badge (1-9) in the top-left corner when a project is selected and not in edit mode.

| State | Style |
|-------|-------|
| Default | absolute top-1 left-1, text-[10px] font-semibold, text-muted-foreground/40 |
| Card hover | text-muted-foreground/70 |
| Card focused | text-muted-foreground/70 |
| No project selected | hidden |
| Edit mode | hidden |

Maximum 9 badges. Cards beyond position 9 show no badge.

#### Roving Tabindex Implementation

- Sidebar: only the currently focused project has `tabindex="0"`, all others have `tabindex="-1"`
- Cards: only the currently focused card has `tabindex="0"`, all others have `tabindex="-1"`
- Arrow keys move the "0" tabindex and call `.focus()` on the new element

#### Focus Visual (per D-14)

All focus states use the existing `focus-visible:ring-2 focus-visible:ring-ring` pattern from button.tsx.

#### Initial Focus (per D-16)

On app mount: if projects exist, focus the first project in sidebar. If no projects, focus the "添加项目" button. Implemented via `useEffect` + `ref.focus()` after loading completes.
