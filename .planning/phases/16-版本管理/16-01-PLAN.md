  <action>
    **Step 1: 修改 Cargo.toml** — 在 [dependencies] 段添加三个新依赖（per RESEARCH.md）:
    - `reqwest = { version = "0.13", features = ["blocking", "json"] }`
    - `semver = "1"`
    - `open = "5"`

    **Step 2: 创建 src-tauri/src/commands/update.rs** — 新文件，包含:

    (a) `UpdateCheckResult` 结构体（derive Serialize）:
    - `has_update: bool`
    - `latest_version: Option<String>`

    (b) `check_for_updates(app: tauri::AppHandle) -> Result<UpdateCheckResult, String>` command:
    - 使用 `use tauri_plugin_store::StoreExt` 打开 "easypack-store.json"
    - 从 `app.config().version` 读取当前版本号（`config.version.as_deref().unwrap_or("0.0.0")`），用 `semver::Version::parse` 解析
    - 检查缓存：读取 store 中 `updateCheck.lastCheckTime`（u64 UNIX timestamp），与当前时间比较，小于 86400 秒（24h）则命中缓存
    - 缓存命中：读取 `updateCheck.latestVersion`，用 semver 比较返回结果
    - 缓存未命中：用 `reqwest::blocking::Client::new()` 请求 `https://api.github.com/repos/armok-c/EasyPack/releases/latest`，设置 `User-Agent: EasyTalk` header 和 10 秒超时
    - 解析响应 JSON 中 `tag_name` 字段，strip "v" 前缀（`strip_prefix('v').unwrap_or(tag_name)`），用 semver 解析
    - 将 `latest > current` 的结果写入 store（`updateCheck.lastCheckTime` + `updateCheck.latestVersion`），调用 `store.save()`
    - 错误处理：404（无 release）返回 `has_update: false`；网络错误/限流静默返回 `has_update: false`；semver parse 失败返回 Err

    (c) `open_release_page() -> Result<(), String>` command:
    - 调用 `open::that("https://github.com/armok-c/EasyPack/releases/latest")`
    - 错误映射为 String

    **Step 3: 修改 src-tauri/src/commands/mod.rs** — 添加 `pub mod update;`

    **Step 4: 修改 src-tauri/src/lib.rs** — 在 `invoke_handler` 宏中添加:
    - `commands::update::check_for_updates`
    - `commands::update::open_release_page`

    **重要注意事项:**
    - reqwest blocking client 在 Tauri command 中是安全的——Tauri 为每个 invoke 在独立线程执行
    - tag_name 返回 "v1.0.0" 格式，必须 strip "v" 前缀再传给 semver::Version::parse
    - store key 使用 `updateCheck.` 前缀避免与现有 key 冲突
    - 使用 `saturating_sub` 做时间差计算，防止整数溢出

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Rust backend -> GitHub API | 外部 HTTP 请求，响应数据不可信 |
| Store 缓存 -> check_for_updates | 本地持久化数据，理论上可被用户篡改但无安全影响 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-01 | Tampering | GitHub API response parsing | mitigate | semver::Version::parse 严格校验 tag_name，非合法版本号直接 fallback 为"无更新"，不 panic |
| T-16-02 | Tampering | MITM on GitHub API | mitigate | reqwest 默认 HTTPS + 证书验证（rustls TLS），无需额外配置 |
| T-16-03 | Denial of Service | API rate limit (60 req/hr) | accept | 24h 缓存确保正常使用不会超限；超限静默返回"无更新"不影响 UX |
| T-16-04 | Information Disclosure | Network error messages | accept | 错误信息只在 Rust 端作为 Err 返回，前端 catch 后静默忽略，不展示给用户 |
</threat_model>
