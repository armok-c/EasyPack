    ```typescript
    interface UseEdgeDrawerOptions {
      visibility: VisibilityState;
      hideToDrawer: () => void;
      showFromDrawer: () => void;
      drawerEnabled: boolean;
    }

    interface UseEdgeDrawerReturn {
      handleDragEnd: () => Promise<void>;       // 拖拽结束时检测吸附
      handleMouseEnter: () => void;              // 取消延迟收回
      handleMouseLeave: () => void;              // 启动延迟收回
      handleDragWhileSnapped: (deltaX: number, deltaY: number) => void; // 取消吸附检测
      isDrawerAnimating: boolean;                // 动画进行中
      snapEdge: SnapEdge | null;                 // 当前吸附边
    }

    export function useEdgeDrawer(options: UseEdgeDrawerOptions): UseEdgeDrawerReturn;
    ```

    内部状态和 refs：
    - `snapEdgeRef: useRef<SnapEdge | null>(null)` -- 当前吸附边
    - `originalRectRef: useRef<Rect | null>(null)` -- 吸附前的窗口位置和尺寸 (D-15)
    - `operationLock: useRef<Promise<void>>(Promise.resolve())` -- mutex 防止动画竞争 (复用 useFloatWindow 模式)
    - `hideTimeoutRef: useRef<ReturnType<typeof setTimeout> | null>(null)` -- 延迟收回定时器 (D-10)
    - `isAnimating: useState(false)` -- 动画进行中标志
    - `currentSnapEdge: useState<SnapEdge | null>(null)` -- 暴露给 UI

    handleDragEnd 实现：
    1. 如果 drawerEnabled === false，直接返回
    2. 如果 visibility !== "VISIBLE"，直接返回
    3. 获取 appWindow.outerPosition(), innerSize(), isMaximized(), primaryMonitor()
    4. 构建 WindowInfo 和 MonitorInfo，调用 detectSnapEdge()
    5. 如果返回 null，直接返回
    6. 保存 originalRectRef（当前窗口位置和尺寸）
    7. 调用 appWindow.setMinSize(new LogicalSize(0, 0)) 移除最小尺寸限制
    8. 计算 sliverRect = calculateSliverRect(edge, workArea, scaleFactor)
    9. 通过 operationLock 序列化：调用 animateWindow 收缩为 sliver
    10. 设置 snapEdgeRef = edge, currentSnapEdge = edge
    11. 调用 hideToDrawer()
    12. emit("drawer:start-polling", { edge, sliverRect }) 通知 Rust 启动轮询

    drawer:mouse-near-edge 事件监听 (useEffect)：
    1. listen("drawer:mouse-near-edge") 
    2. 如果 visibility === "DRAWER_HIDDEN" 且 originalRectRef 有值
    3. emit("drawer:stop-polling") 停止轮询
    4. 通过 operationLock 序列化：调用 animateWindow 从 sliver 展开到 originalRectRef 位置和尺寸
    5. 调用 showFromDrawer()
    6. 注意：此时 snapEdgeRef 保持不变（窗口仍"吸附"，只是滑出显示）

    handleMouseLeave (D-10 延迟收回)：
    1. 如果 snapEdgeRef 为 null，返回
    2. hideTimeoutRef = setTimeout(() => { slideIn(); }, 400) (取 400ms 中间值)
    3. slideIn 函数：animateWindow 从当前位置收缩为 sliver，然后 hideToDrawer()，emit("drawer:start-polling")

    handleMouseEnter：
    1. 如果 hideTimeoutRef 有值，clearTimeout 并设为 null

    handleDragWhileSnapped (D-14 取消吸附)：
    1. 如果 snapEdgeRef 为 null，返回
    2. 计算 delta = Math.abs(deltaX) + Math.abs(deltaY)
    3. 如果 delta > 20 (D-14 阈值)
    4. 取消吸附：emit("drawer:stop-polling")
    5. 恢复窗口到 originalRectRef 位置（如果当前不在那里）
    6. 调用 appWindow.setMinSize(new LogicalSize(600, 400)) 恢复最小尺寸限制
    7. 设置 snapEdgeRef = null, currentSnapEdge = null
    8. 如果 visibility !== "VISIBLE"，调用 showFromDrawer()

    animateWindow 调用方式：
    - from/to 为 AnimState 对象
    - onUpdate 中调用 appWindow.setPosition(new LogicalPosition(state.x, state.y)) 和 appWindow.setSize(new LogicalSize(state.w, state.h))
    - durationMs = 200 (D-03, D-11)

    清理 (useEffect return)：
    - clearTimeout(hideTimeoutRef)
    - unlisten drawer:mouse-near-edge

    ```rust
    // 边缘抽屉：鼠标位置轮询
    // 前端通过 emit("drawer:start-polling", payload) 启动
    // 前端通过 emit("drawer:stop-polling") 停止
    // 定时器回调中检测鼠标是否在 sliver 区域附近，是则 emit("drawer:mouse-near-edge")
    use std::sync::{Arc, Mutex};
    use tauri::{Listener, Emitter, Manager};
    use tauri::dpi::PhysicalPosition;

    // 共享状态：轮询定时器句柄
    let polling_handle: Arc<Mutex<Option<tauri::async_runtime::JoinHandle<()>>>> = Arc::new(Mutex::new(None));
    let sliver_rect: Arc<Mutex<Option<(f64, f64, f64, f64)>>> = Arc::new(Mutex::new(None)); // (x, y, w, h) 物理坐标

    let app_handle = app.handle().clone();
    let ph = polling_handle.clone();
    let sr = sliver_rect.clone();
    app.listen("drawer:start-polling", move |event| {
        // 解析 payload: { sliverRect: { x, y, w, h } } (逻辑坐标)
        // 获取 scaleFactor，转为物理坐标存入 sliver_rect
        // 启动定时器：每 100ms 调用 app.cursor_position()
        // 比较 cursor 物理坐标是否在 sliver 物理 rect ± 5px 内
        // 如果在范围内：emit("drawer:mouse-near-edge")，停止轮询
    });

    let ph2 = polling_handle.clone();
    app.listen("drawer:stop-polling", move |_| {
        // 停止轮询定时器
    });
    ```

    具体实现细节：

    1. `drawer:start-polling` 事件：
       - Payload 格式：`{ "sliverRect": { "x": number, "y": number, "w": number, "h": number } }`（逻辑像素坐标）
       - 获取 primaryMonitor 的 scaleFactor
       - 将逻辑坐标转为物理坐标：`phys_x = logical_x * scaleFactor`
       - 存入 `sliver_rect` Mutex
       - 取消已有定时器（如果存在）
       - 启动新的 tokio::spawn 循环，每 100ms：
         - 调用 `app_handle.cursor_position()` 获取 PhysicalPosition
         - 从 sliver_rect 读取物理坐标
         - 判断 cursor 是否在 sliver rect 扩展 5px 范围内
         - 如果在范围内：emit("drawer:mouse-near-edge", {})，停止自身

    2. `drawer:stop-polling` 事件：
       - 取消定时器句柄（JoinHandle::abort()）
       - 清空 sliver_rect

    3. 注意使用 `app.handle().clone()` 避免借用问题

    **注意 cursor_position() API**: 检查 Tauri v2 AppHandle 是否有 cursor_position() 方法。如果没有，使用 Win32 API 替代：
    ```rust
    #[cfg(target_os = "windows")]
    fn get_cursor_pos() -> Option<(i32, i32)> {
        use std::mem;
        let mut point: winapi::um::winnt::POINT = unsafe { mem::zeroed() };
        let result = unsafe { winapi::um::winuser::GetCursorPos(&mut point) };
        if result != 0 { Some((point.x, point.y)) } else { None }
    }
    ```
    如果走 Win32 路径，在 Cargo.toml 添加 `winapi = { version = "0.3", features = ["winuser"] }` 依赖。
