  - Test 2: build_bat_content 对简单多行命令（is_batch_script=false, strict=false）用 " & " 连接
  - Test 3: build_bat_content 对批处理脚本（is_batch_script=true）原样写入内容，头部仍包含 chcp + cd /d

  <action>
  在 Cargo.toml [dependencies] 中添加 `tempfile = "3"`。

  在 shell.rs 中添加:

  1. `build_bat_content(project_path: &str, script_lines: &str, is_batch_script: bool, strict: bool) -> String` 函数:
     - 头部始终: `@echo off\nchcp 65001 >nul\ncd /d "{project_path}"` (per D-10, D-11)
     - is_batch_script=true 时: 原样追加 script_lines
     - is_batch_script=false 时: 过滤空行和仅空白行，用 separator (strict ? " && " : " & ") 连接后追加 (per D-05, D-08)
     - 空内容时只写头部不崩溃

  2. `execute_script(project_path: String, script_content: String, is_batch_script: bool, strict: bool) -> Result<String, String>` Tauri command:
     - 调用 build_bat_content 生成 .bat 内容
     - 使用 `tempfile::Builder::new().prefix("easypack-").suffix(".bat").rand_bytes(8).tempfile_in(std::env::temp_dir())` 创建临时文件 (per D-09，不删除，放系统 temp 目录)
     - 注意：tempfile::Builder 创建 NamedTempFile，需要 .into_temp_path() 然后 .keep() 来保留文件（不自动删除）
     - 用 std::fs::write 写入 .bat 内容到路径
     - 执行命令: `cmd /C start "" /d "{project_path}" cmd /K "{bat_path}"` (复用 raw_arg 模式)
     - 返回 .bat 文件路径字符串

  3. 为 execute_script 添加 #[cfg(test)] 单元测试:
     - test_build_bat_content_strict: 验证 && 连接 + 头部包含 chcp + cd /d
     - test_build_bat_content_lenient: 验证 & 连接
     - test_build_bat_content_batch_script: 验证原样写入
     - test_build_bat_content_empty_lines: 验证空行过滤
     - test_execute_script_creates_bat_file: 验证 .bat 文件创建且路径包含 "easypack-"

  关键注意事项:
  - tempfile crate 的 NamedTempFile 在 drop 时会自动删除文件。使用 `.into_temp_path().keep()` 来保留文件不删除 (per D-09)。
  - bat_path 传给 cmd.exe 时必须用双引号包裹（路径可能含空格或中文）
  - 继续使用 raw_arg() 模式（已在文件顶部 import）
  - .bat 文件写入时使用 UTF-8 编码（std::fs::write 默认行为），配合 chcp 65001 确保中文路径兼容

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 前端 -> Rust execute_script | script_content 跨越 IPC 边界，由用户输入 |
| Rust -> 文件系统 | .bat 文件写入系统 temp 目录 |
| Rust -> cmd.exe | 执行外部进程 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-01 | Tampering | execute_script script_content | accept | 脚本内容由用户完全控制（本地桌面应用），不是外部输入。用户编写什么就执行什么 |
| T-17-02 | Denial of Service | temp 目录 .bat 文件积累 | accept | 临时文件放系统 temp 目录，随 OS 清理机制回收 (per D-09)。单用户场景积累速度远低于磁盘容量 |
| T-17-03 | Tampering | project_path 参数 | mitigate | 路径来自用户文件夹选择（dialog），execute_script 中 cd /d 用引号包裹防止路径注入。与现有 execute_command 一致 |
</threat_model>
