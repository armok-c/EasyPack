### 3. Custom Command Visual Marker

**决策 (Claude's Discretion)：** 自定义指令卡片使用细微的左侧边框颜色差异标记

| Property | Value |
|----------|-------|
| 标记方式 | 左侧 2px 彩色边框 (`border-l-2 border-l-blue-400/50`) |
| 预设指令 | 无额外标记（仅 `border border-white/10`） |
| 自定义指令 | `border border-white/10 border-l-2 border-l-blue-400/50` |
| 颜色 | `blue-400/50` — 与 accent 色系区分，但不会过于突兀 |
| hover 时 | `border-l-blue-400/70` (增强可见度) |

**Why this approach:** 左侧彩色边框是轻量级的视觉区分方式，不破坏卡片整体一致的圆角风格，且不会占据卡片内部空间。类似 VS Code 的 tab modified indicator 设计语言。

### 4. Edit Mode Card Styles

**编辑模式下的自定义指令卡片：**

| 预设指令 | 无删除按钮，无编辑弹窗触发，点击无反应 (per D-04) |

## ContextMenu Specification

### 右键菜单项

**仅对自定义指令卡片生效 (per D-05)：**

```
┌────────────────────┐
│  编辑              │
│  ─────────────     │
│  删除              │
└────────────────────┘
```

| Property | Value |
|----------|-------|
| Trigger | 右键点击自定义指令卡片 |
| 预设指令右键 | 无菜单（不注册 ContextMenu） |
| Menu bg | shadcn default (`bg-popover`) |
| Menu border | `border border-white/10` |
| Menu radius | `rounded-md` (shadcn default) |
| Item padding | shadcn default |
| "编辑" onClick | 打开 CommandDialog，预填当前指令数据 |
| "删除" onClick | 直接删除，无确认弹窗 (per D-03) |
| "删除" 样式 | `text-red-400 focus:text-red-300` (destructive variant) |

**嵌套规则：** Dialog 必须包裹 ContextMenu（per shadcn/ui 官方文档和 RESEARCH.md Pattern 1）

```tsx
<Dialog>
  <ContextMenu>
    <ContextMenuTrigger asChild>
      <CommandCard ... />
    </ContextMenuTrigger>
    <ContextMenuContent>
      <ContextMenuItem>编辑</ContextMenuItem>
      <ContextMenuItem className="text-red-400">删除</ContextMenuItem>
    </ContextMenuContent>
  </ContextMenu>
  <DialogContent>...</DialogContent>
</Dialog>
```

interface CommandItem {
