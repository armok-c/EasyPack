# Plan 20-01: Store 层重构 + Profile 管理 + 数据迁移

将 useProject 从单 store 实例重构为 mainStore（profile 元信息 + 全局设置）+ profileStore（当前活跃 profile 数据）双实例架构。实现首次启动自动迁移、Profile CRUD、switchProfile（含并发安全）、import/export。所有 CRUD 操作改为使用 profileStore。对外暴露完整 Profile 管理接口，供 Plan 02 的 UI 层消费。

2. **实现 switchProfile(id: string)：**
   ```typescript
   const switchProfile = useCallback(async (id: string) => {
     if (!mainStore) return;
     if (switchingProfileRef.current) return; // 并发安全 per D-24
     if (id === activeProfileId) return;       // 跳过重复切换

     switchingProfileRef.current = true;
     setProfileSwitching(true);

     try {
       // 1. 更新主 store 的 activeProfileId
       await mainStore.set(ACTIVE_PROFILE_KEY, id);
       await mainStore.save();

       // 2. 加载新 profile store
       const ps = await load(profileStorePath(id), { autoSave: 100, defaults: {} });

       // 3. 批量重置所有 React state per D-04/D-05
       //    - commandMode → "global"
       //    - editMode → false
       //    - 其余从新 profile store 读取
       setCommandMode("global");
       setEditMode(false);
       loadProfileDataIntoState(ps);

       // 4. 更新 profileStore 引用
       setProfileStore(ps);
       // 5. 同步更新 store 引用（供 useRecentCommands 等）
       setStore(ps);
       // 6. 更新 activeProfileId
       setActiveProfileId(id);
       // 7. 重置 project info
       setProjectInfo(null);
     } finally {
       switchingProfileRef.current = false;
       setProfileSwitching(false);
     }
   }, [mainStore, activeProfileId]);
   ```

### Task 5: Profile CRUD 方法（create/delete/rename）

- src/lib/types.ts — ProfileMeta 类型定义

<action>
在 useProject.ts 中实现三个 Profile 管理方法：

1. **createProfile(name: string)：**
   ```typescript
   const createProfile = useCallback(async (name: string) => {
     if (!mainStore) return;
     const id = crypto.randomUUID();
     const meta: ProfileMeta = { id, name, createdAt: Date.now() };
     const updated = [...profileMetas, meta];
     // 创建空的 profile store 文件
     const ps = await load(profileStorePath(id), { autoSave: 100, defaults: {} });
     await ps.save();
     // 更新主 store
     await mainStore.set(PROFILES_KEY, updated);
     await mainStore.save();
     setProfileMetas(updated);
     // 自动切换到新 profile
     await switchProfile(id);
   }, [mainStore, profileMetas, switchProfile]);
   ```

2. **deleteProfile(id: string)：**
   - 不允许删除最后一个 profile（至少保留一个）
   - 从 profileMetas 中移除目标
   - 更新主 store 的 profiles 列表
   - 如果删除的是当前活跃 profile，自动切换到列表中第一个
   - 删除对应的 profile JSON 文件（通过 tauri-plugin-fs 或 store.delete 不可行时标记忽略）
   - 注意：tauri-plugin-store 没有删除文件 API，删除的 profile 文件可以保留在磁盘（无害），或者使用 `@tauri-apps/plugin-fs` 的 `remove` 删除

3. **renameProfile(id: string, newName: string)：**
   - 更新 profileMetas 中对应条目的 name
   - 更新主 store 的 profiles 列表
   - 不需要修改 profile store 文件（name 只存在主 store 的 metas 中）

4. **在 return 对象中暴露：** `createProfile`、`deleteProfile`、`renameProfile`
</action>
