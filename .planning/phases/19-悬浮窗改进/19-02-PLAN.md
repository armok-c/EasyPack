# Plan 19-02: 项目切换通信 + useFloatWindow 扩展

实现折叠态项目循环切换（点击名称 → 下一个项目）+ 主窗口/悬浮窗项目双向同步 + 折叠/展开时窗口物理尺寸配合。核心改造：扩展 syncState 推送完整项目列表、新增 float:switch-project 事件通信、添加窗口 resize 能力。

## Tasks

### Task 1: 扩展 useFloatWindow — 推送项目列表 + resize + 监听项目切换

<read_first>
- src/hooks/useFloatWindow.ts — 当前 292 行，syncState 推送 `{ project, commands }`，需扩展为 `{ project, projects, commands }`；需新增 resizeFloat 方法和 float:switch-project 监听
- src/hooks/useProject.ts:47-76 — useProject 接口，提供 projects 数组、selectProject、selectedId 等
- src/App.tsx:148-152 — useFloatWindow 的调用点，传入 `{ currentProject, commands, onExecute }`，需新增 `projects` 和 `selectProject` 参数
</read_first>

<action>
**1. 扩展 useFloatWindow 参数接口：**
- UseFloatWindowOptions 新增字段：
  - `projects: ProjectItem[]` — 完整项目列表
  - `onSwitchProject: (projectId: string) => void` — 主窗口收到切换请求时的回调
- 为 `projects` 和 `onSwitchProject` 添加 ref 模式（projectsRef、onSwitchProjectRef），遵循现有 currentProjectRef/commandsRef/onExecuteRef 的模式

**2. 扩展 syncState 推送：**
- syncState 中的 emitTo payload 从 `{ project, commands }` 改为 `{ project, projects, commands }`
- projectsRef.current 即为新字段值

**3. 新增 resizeFloat 方法：**
- 返回值 UseFloatWindowReturn 新增 `resizeFloat: (width: number, height: number) => Promise<void>`
- 实现：获取 floatWindowRef.current，调用 `win.setSize(new LogicalSize(width, height))`
- 从 `@tauri-apps/api/dpi` import LogicalSize
- resize 操作包裹在 operationLock mutex 中，与现有 toggleFloat/destroyFloat 一致
- resize 后不需要调整位置（per CONTEXT.md Claude's Discretion 关于位置的处理）

**4. 新增 float:switch-project 事件监听：**
- 在 registerFloatListeners 中新增 listen `"float:switch-project"`
- 事件 payload 类型为 `{ projectId: string }`
- 监听回调调用 `onSwitchProjectRef.current(event.payload.projectId)`
- unlisten 推入 unlistenersRef

**5. 自动同步：**
- useEffect 依赖数组中新增 `projects`，确保 projects 变化时也触发 syncState
- 当前已有 `[currentProject, commands, floatVisible, syncState]`，添加 projects

**6. 窗口创建时调整约束：**
- createFloat 中 `minWidth` 从 220 改为 130（允许折叠态尺寸）
- `maxWidth` 移除或设为 220（允许展开态尺寸）
- `resizable` 保持 false（通过代码控制 resize，不允许用户手动拖拽调整）

**7. positionFloatTopRight 调整：**
- 当前硬编码 `const posX = logicalRight - 220 - 16`
- 这里的 220 是窗口宽度。不需要修改，因为初始创建时仍是 220px 展开态
- 但如果需要更灵活，可以在后续考虑

</action>

### Task 2: App.tsx 集成 + FloatApp 项目切换逻辑 + resize 通信

<read_first>
- src/App.tsx:148-152 — useFloatWindow 调用点，需传入新参数 projects 和 onSwitchProject
- src/App.tsx:33-72 — useProject 解构，包含 projects、selectProject 等需要的字段
- src/components/FloatApp.tsx — Task 1 of Plan 01 完成后的文件，包含折叠/展开 UI，需添加 projects state、项目切换逻辑、resize 通信
- src/hooks/useProject.ts:11-18 — ProjectItem 接口定义（id, name, path, addedAt, icon?, color?）
</read_first>

<action>
**1. App.tsx 集成（3 处改动）：**

a) 修改 useFloatWindow 调用，添加新参数：
   - `projects` — 从 useProject 解构已有
   - `onSwitchProject` — 使用 `selectProject` 回调（useProject 已解构），包装为 `(projectId: string) => void` 即可

b) 从 useFloatWindow 返回值解构新增的 `resizeFloat`

c) 无需额外 useEffect 监听 float:switch-project——因为监听已在 useFloatWindow 的 registerFloatListeners 中完成，App 只需传入 onSwitchProject 回调

**2. FloatApp.tsx 改造（4 处改动）：**

a) 添加 projects state：
   - `const [projects, setProjects] = useState<ProjectItem[]>([])`
   - 修改 float:state-update 监听，payload 类型新增 `projects: ProjectItem[]`
   - 在 listener 中 `setProjects(event.payload.projects)`

b) 折叠态项目名称点击逻辑（per D-08/D-11）：
   - 在折叠态的项目名称 span 上添加 onClick handler
   - handler 逻辑：
     1. 如果没有 project 或 projects 为空，return（无操作）
     2. 如果 projects 只有一个项目，return（无需切换）
     3. 计算当前项目在 projects 中的 index：`projects.findIndex(p => p.id === project.id)`
     4. 下一个 index：`(currentIndex + 1) % projects.length`（循环切换，per D-11）
     5. emit `"float:switch-project"` 事件，payload `{ projectId: projects[nextIndex].id }`
   - 名称 span 需要 `cursor-pointer` 样式指示可点击
   - 名称 click 需要 `e.stopPropagation()` 防止触发拖拽

c) 折叠/展开时的 resize 通信：
   - 导入 `LogicalSize` from `@tauri-apps/api/dpi`
   - 添加 `resizeWindow` 辅助函数：使用 `getCurrentWindow().setSize(new LogicalSize(width, height))` 在悬浮窗端直接 resize
   - 折叠时：先播放 CSS 过渡（setCollapsed(true)），然后用 `setTimeout(300)` 等待 CSS 动画完成后调用 `resizeWindow(130, 32)`
   - 展开时：先 resize 到展开尺寸 `resizeWindow(220, 400)`，然后在 requestAnimationFrame 后 `setCollapsed(false)` 让 CSS 展开动画播放
   - 展开时高度计算：根据当前 commands 数量动态计算，`32 + Math.min(commands.length * 28 + 48, 400)` 或简单使用固定的 300（与创建时一致）

d) 确保 float:state-update 监听器中 projects 字段被正确接收：
   - 更新 listen 的泛型参数，payload 类型从 `{ project: ProjectItem | null; commands: CommandItem[] }` 改为 `{ project: ProjectItem | null; projects: ProjectItem[]; commands: CommandItem[] }`

</action>
