  <action>
    **修改 `src/components/ui/dialog.tsx` 的 DialogContent 组件（第 62 行的 className）：**

    将现有 className 从:
    ```
    "fixed top-[50%] left-[50%] z-50 grid w-full max-w-[calc(100%-2rem)] translate-x-[-50%] translate-y-[-50%] gap-4 rounded-lg border bg-background p-6 shadow-lg duration-200 outline-none data-[state=closed]:animate-out data-[state=closed]:fade-out-0 data-[state=closed]:zoom-out-95 data-[state=open]:animate-in data-[state=open]:fade-in-0 data-[state=open]:zoom-in-95 sm:max-w-lg"
    ```

    改为:
    ```
    "fixed top-[50%] left-[50%] z-50 flex flex-col w-full max-w-[calc(100%-2rem)] max-h-[90vh] translate-x-[-50%] translate-y-[-50%] gap-4 rounded-lg border bg-background p-6 shadow-lg duration-200 outline-none data-[state=closed]:animate-out data-[state=closed]:fade-out-0 data-[state=closed]:zoom-out-95 data-[state=open]:animate-in data-[state=open]:fade-in-0 data-[state=open]:zoom-in-95 sm:max-w-lg"
    ```

    变更点（共 3 处）:
    1. `grid` → `flex flex-col` (per D-10: flex 布局支持固定 header/footer + 滚动内容区)
    2. 添加 `max-h-[90vh]` (per D-09: 全局最大高度限制，90vh 来自 UI-SPEC 的决定)

    **关键：不删除 `gap-4`**。虽然从 grid 改为 flex，gap-4 在 flex 布局中仍然有效，会在子元素之间添加间距。保持 gap-4 确保现有对话框的视觉间距不变。

    **不需要修改 DialogHeader、DialogFooter、DialogDescription 等子组件**。原因:
    - 现有的 DialogContent 子组件（DialogHeader、中间内容、DialogFooter）已经是 `<div>` 元素
    - flex-col 布局下它们会自动按顺序排列
    - 当内容超出 max-h-[90vh] 时，浏览器会自动裁剪（因为 DialogContent 本身 overflow 默认 visible）
    - 具体的 `overflow-y-auto` 应该由使用 DialogContent 的各组件在需要滚动的内容区域自行添加
    - 这样更灵活，不需要强制所有对话框都改变内部结构

    **关于关闭按钮定位**:
    现有关闭按钮使用 `absolute top-4 right-4`，改为 flex-col 后 absolute 定位仍然有效（flex 容器默认 position: relative 的定位上下文不变），不需要修改。

    **验证现有组件兼容性**:
    - ProjectSettingsDialog: 使用 DialogContent -> DialogHeader -> div.py-4 -> DialogFooter 结构。flex-col 布局下，header、内容、footer 仍按顺序排列。中间的 `div.py-4` 如有内容溢出，由 ProjectSettingsDialog 本身在需要时添加 overflow 处理（Phase 8 Plan 03 会处理）
    - 其他使用 DialogContent 的组件同理

    **注意 Pitfall 2 (RESEARCH.md):** min-h-0 是在 flex 子元素需要缩小以触发 overflow 时才需要的。此处 DialogContent 本身是外层容器，它通过 max-h-[90vh] 限制高度。具体内容区的滚动由各对话框自行实现（Plan 03 的 ProjectSettingsDialog 会在内容区添加 overflow 处理）。
