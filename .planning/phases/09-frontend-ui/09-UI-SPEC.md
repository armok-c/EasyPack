## Interaction Specifications

### 1. Command Mode Toggle Group (UI-10, per D-01, D-02, D-05, D-06)

#### Visual Layout

Replace the existing text link switcher (MainArea.tsx lines 207-228) with a single-row button bar:

```
+--[Button Row: flex items-center justify-between mt-2]-----------+
|                                                                  |
|  +--[Toggle Group: inline-flex rounded-md overflow-hidden]--+   |
|  | [全局指令(secondary)] [项目指令(ghost)]                     |   |
|  +-----------------------------------------------------------+   |
|                                                                  |
|                           [打开文件夹(outline)]                   |
|                                                                  |
+------------------------------------------------------------------+
```

#### Toggle Group Implementation Decision

**Approach: Hand-styled Button pair, NOT shadcn ToggleGroup component.**

Rationale:
- Only 2 buttons in a mutually exclusive toggle — shadcn ToggleGroup adds unnecessary Radix abstraction for this simple case
- D-01 requires "紧邻拼合" (flush-joined) visual — needs custom border-radius removal that's easier with plain Buttons
- Uses existing shadcn Button variants (secondary/ghost) directly, no new component installation
- Fewer dependencies, smaller bundle, simpler code

#### Toggle Group CSS Specification

```
Container: inline-flex rounded-md overflow-hidden border border-white/10
  - rounded-md: outer group has rounded corners
  - overflow-hidden: clips inner buttons to the group border-radius
  - border: subtle border unifies the two buttons visually

Active button (secondary variant):
  - Remove right border-radius: rounded-l-none (if left button) or rounded-r-none (if right button)
  - Uses shadcn secondary variant styles (bg-secondary, text-secondary-foreground)

Inactive button (ghost variant):
  - Remove left border-radius: rounded-r-none (if left button) or rounded-l-none (if right button)
  - Uses shadcn ghost variant styles (transparent bg, text-muted-foreground)

Divider between buttons:
  - border-r border-white/10 on the left button
  - Provides visual separation without a gap
```

#### Toggle Group States

| State | Active Button Style | Inactive Button Style |
|-------|-------------------|----------------------|
| Global mode active | "全局指令" button = secondary variant | "项目指令" button = ghost variant |
| Project mode active | "项目指令" button = secondary variant | "全局指令" button = ghost variant |
| Project has no custom commands | (Global mode auto-active per useProject.ts line 88) | "项目指令" button = ghost + disabled:opacity-50 + pointer-events-none (D-05) |

#### Toggle Group Interaction (per D-06)

| Event | Behavior |
|-------|----------|
| Click "全局指令" when in project mode | Calls `disableProjectCommands()`, switches to global |
| Click "项目指令" when in global mode | Calls `enableProjectCommands()`, switches to project |
| Click currently active button | No-op (already in that mode) |
| Click "项目指令" while disabled | No-op (D-05: disabled state prevents interaction) |
| Keyboard: Tab | Moves focus between Toggle Group and "打开文件夹" button |
| Keyboard: Enter/Space | Activates the focused button |
| Keyboard: Arrow keys | Not used (not a radio group pattern — two separate buttons) |

#### Disabled State Logic (D-05)

```typescript
// Derived from useProject.ts existing logic
const hasProjectCommands = selectedId
  ? (projectCommandsMap[selectedId]?.length ?? 0) > 0
  : false;

// "项目指令" button disabled when:
// 1. No project is selected, OR
// 2. Selected project has no project-level command set
const isProjectToggleDisabled = !selectedId || !hasProjectCommands;
```

### 2. "打开文件夹" Button (PROJ-11, per D-03, D-04)

#### Visual Specification

```
+--[Button: outline variant, sm size, icon+text]--+
|  [FolderOpen 14px]  打开文件夹                    |
+--------------------------------------------------+
```

- Variant: `outline` — visible border distinguishes it from Toggle Group (which uses secondary/ghost)
- Size: `sm` (h-8, text-xs) — matches Toggle Group height (D-02)
- Icon: `FolderOpen` from lucide-react, `size-3.5` (14px)
- Alignment: right side of button row via `justify-between` on parent flex container

#### Hover/Active States

| State | Style |
|-------|-------|
| Default | border border-input, bg-background — standard outline appearance |
| Hover | bg-accent, text-accent-foreground — shadcn outline hover (from button.tsx line 17) |
| Active (mousedown) | scale-[0.98] transition-transform — subtle press feedback |
| Focus (keyboard) | ring-2 ring-ring ring-offset-2 — standard shadcn focus ring |
| Disabled (no project) | disabled:opacity-50 disabled:pointer-events-none — standard shadcn disabled |

#### Rust Backend: open_folder Command (per D-04)

```rust
// In src-tauri/src/commands/shell.rs (reuse existing file, no new module needed)

#[tauri::command]
pub async fn open_folder(path: String) -> Result<(), String> {
    StdCommand::new("explorer.exe")
        .raw_arg(format!("\"{}\"", path))
        .spawn()
        .map_err(|e| format!("Failed to open folder: {}", e))?;
    Ok(())
}
```

Registration in `src-tauri/src/lib.rs`:
```rust
commands::shell::open_folder,  // add to invoke_handler generate_handler! macro
```

#### Frontend Integration

```typescript
// In useProject.ts — new function
const openFolder = useCallback(async (path: string) => {
  try {
    await invoke("open_folder", { path });
  } catch (error) {
    toast.error("无法打开文件夹", {
      description: "路径无效或文件夹不存在"
    });
  }
}, []);

// In App.tsx — pass as prop
<MainArea
  ...existing props
  onOpenFolder={() => openFolder(currentProject.path)}
/>

// In MainArea.tsx — consume prop
interface MainAreaProps {
  ...existing props
  onOpenFolder: () => void;
}
```
