#### State Management Logic

```typescript
// New state in CommandDialog
const [selectedCategory, setSelectedCategory] = useState<string>("");
const [selectedPresetId, setSelectedPresetId] = useState<string>("");

// Derived: filter presets by selected category
const filteredPresets = useMemo(
  () => ALL_PRESETS.filter((p) => p.category === selectedCategory),
  [selectedCategory]
);

// Category change: reset preset selection + form fields
const handleCategoryChange = useCallback((categoryId: string) => {
  setSelectedCategory(categoryId);
  setSelectedPresetId("");
  // Do NOT clear name/command/icon — user may have manually edited them
}, []);

// Preset change: auto-fill form fields
const handlePresetChange = useCallback((presetId: string) => {
  const preset = ALL_PRESETS.find((p) => p.id === presetId);
  if (preset) {
    setName(preset.name);
    setCommand(preset.command);
    setSelectedIcon(preset.icon);
    setNameDirty(false);
    setCommandDirty(false);
  }
  setSelectedPresetId(presetId);
}, []);

// Manual field edit behavior (Claude's Discretion):
// When user manually edits name or command, clear preset selection indicator
// but keep the visual state of the selects (category/command still shown).
// This avoids jarring UX where selects reset while user is typing.
// The form submits whatever is currently in name/command/icon fields,
// regardless of preset selection state.
```

#### Select Interaction States

| Event | Behavior |
|-------|----------|
| Click category Select trigger | Opens dropdown with 4 categories: Git, NPM/Node, Python/Pip, Rust/Cargo |
| Select a category | Dropdown closes; command Select becomes enabled; command Select filters to that category's presets |
| Click command Select (no category selected) | No dropdown opens; Select is `disabled` |
| Select a command preset | Dropdown closes; name, command, icon fields auto-fill with preset values |
| Change category after command selected | Command Select resets (clears value); name/command/icon fields reset to empty (new category = new selection) |
| Manually edit auto-filled name field | Preset selection remains in selects (visual reference); field becomes user-edited |
| Manually edit auto-filled command field | Same as above |
| Manually select a different icon | Preset selection remains in selects; icon overrides preset default |
| Open dialog in edit mode | Both selects show empty/placeholder state (editing existing command, not preset selection) |
| Submit form | Uses current name/command/icon field values, ignores preset state |

#### Data Change in presets.ts

```typescript
// OLD: PRESET_COMMANDS had 4 items
// NEW: getDefaultsAsCommandItems() returns only 2

export function getDefaultsAsCommandItems(): CommandItem[] {
  return [
    {
      id: "preset-git-pull",
      name: "Git Pull",
      command: "git pull",
      icon: "GitBranch",
      type: "preset" as const,
      scope: "global" as const,
      addedAt: 0,
    },
    {
      id: "preset-claude",
      name: "启动 Claude",
      command: "claude",
      icon: "Sparkles",
      type: "preset" as const,
      scope: "global" as const,
      addedAt: 1,
    },
  ];
}
```

#### Migration Strategy (per STATE.md blocker)

Existing users have `preset-0` through `preset-3` IDs in their store. Migration approach:
1. `getPresetAsCommandItems()` continues to use the 2 new defaults with semantic IDs
2. When loading from store, global commands with old `preset-{n}` IDs are not in the new preset list, so they appear as custom commands or are filtered out
3. `enableProjectCommands()` calls `getPresetAsCommandItems()` to create project command copies — will now copy only the 2 defaults
4. No explicit store migration needed: old preset-0/1 cards simply won't appear in the preset list anymore; users who had them as globals see them vanish and can re-add via preset selector

| Keyboard: Enter/Space | Opens Select dropdown or selects focused item |
| Keyboard: Arrow keys | Navigate Select dropdown items (Radix built-in) |
| Keyboard: Escape | Closes Select dropdown without selection (Radix built-in) |
