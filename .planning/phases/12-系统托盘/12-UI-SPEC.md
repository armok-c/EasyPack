### System Tray Menu (Native, not React)

The tray context menu is a native Tauri Menu — NOT a React component. Its visual rendering is handled by the OS. However, the text content and structure are defined here:

**Menu structure (D-04):**
```
+-------------------------------+
| 显示/隐藏窗口                  |  <- id: "toggle-window"
|-------------------------------|
| ▸ EasyPack (my-project)      |  <- id: "current-project", disabled
| ▸ 执行: npm run build        |  <- id: "cmd-{index}"
| ▸ 执行: npm run dev          |  <- id: "cmd-{index}"
| ... (up to 8 items)          |
|-------------------------------|
| 退出                          |  <- id: "quit"
+-------------------------------+
```

**Menu text rules:**
- Toggle item: "显示窗口" when window is hidden, "隐藏窗口" when window is visible
- Project item: "▸ EasyPack ({project.name})" — disabled state, informational only
- Command items: "▸ 执行: {command.name}" — prefix "▸ 执行: " per D-01
- Quit item: "退出"
- No emoji in any menu item

**Empty state (D-03):** When no recent commands exist, the project name and command list are completely omitted. Menu shows only:
```
+-------------------------------+
| 显示窗口                      |
|-------------------------------|
| 退出                          |
+-------------------------------+
```

**Tray tooltip (D-12):** Fixed text "EasyPack", never changes.

## Interaction Contracts

### Window Close Behavior (D-05, D-06, D-07)

| Trigger | Behavior |
|---------|----------|
| TitleBar X button | `appWindow.hide()` — hide to tray |
| Alt+F4 | Intercepted via `onCloseRequested` → `event.preventDefault()` + `appWindow.hide()` |
| TitleBar minimize | `appWindow.minimize()` — normal minimize to taskbar |
| Tray "退出" | `app.exit(0)` — actual app termination |

### Tray Icon Click (D-09, D-10, D-11)

| Trigger | Behavior |
|---------|----------|
| Left-click | Always show window + focus (never toggle, per D-09) |
| Double-click | No action (prevents conflict with single-click, per D-10) |
| Right-click | Show context menu |

### Settings Switch Dependencies

| Switch | Dependency | Behavior when dependency disabled |
|--------|------------|-----------------------------------|
| "关闭时隐藏到托盘" | "启用系统托盘" must be ON | "关闭时隐藏到托盘" is visually disabled (opacity-50, pointer-events-none) |
| "启用系统托盘" OFF | — | Tray icon disappears immediately, close button restores to `appWindow.close()` (exit) |

### Window Visibility State Machine (D-17, D-18)

```
States: VISIBLE | TRAY_HIDDEN

Transitions:
  VISIBLE     --(close button / Alt+F4)--> TRAY_HIDDEN
  TRAY_HIDDEN --(tray left-click / "显示窗口")--> VISIBLE
  TRAY_HIDDEN --(tray "退出")--> APP_EXIT
```

Phase 14 will extend to: `VISIBLE | TRAY_HIDDEN | DRAWER_HIDDEN` — design interfaces to support this extension.
