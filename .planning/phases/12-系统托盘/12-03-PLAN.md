    **改动 3：tray cleanup 中 await close()**

    Effect 1 cleanup 函数（L176-179）中的 `trayRef.current.close().catch(console.error)` 改为 await 模式。

    将整个 cleanup return 改为 async IIFE 模式：
    ```typescript
    return () => {
      cancelled = true;
      const tray = trayRef.current;
      trayRef.current = null;
      if (tray) {
        tray.close().catch(console.error);
      }
    };
    ```
    注意：cleanup 函数不能是 async 的，保持 .catch(console.error) 模式但先赋值到局部变量再清空 ref，避免异步关闭期间其他代码看到已关闭的 tray。这与旧代码行为一致，只是确保 trayRef.current 在 close 调用前就被清空，防止其他 effect 在异步窗口期使用已关闭的 tray。

    同时修改 Effect 1 中 enabled=false 分支（L121-125）的关闭逻辑也采用同样模式——先清空 ref，再 close：
    ```typescript
    if (!enabled) {
      if (trayRef.current) {
        const tray = trayRef.current;
        trayRef.current = null;
        tray.close().catch(console.error);
      }
      return;
    }
    ```

    **不要修改 Effect 2 的依赖数组**。Effect 2 依赖 `[enabled, currentProject, recentCommands, visibility, commands]` 保持不变——当 currentProject/recentCommands 变化时触发 Effect 2，但 buildMenu 内部从 ref 读取最新值，不再依赖闭包捕获。
