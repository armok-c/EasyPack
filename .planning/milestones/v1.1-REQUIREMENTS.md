## v1.1 Requirements

### FIX — 命令执行修复

- [x] **FIX-01**: 用户点击任何项目的任何指令卡片后，终端能正确在项目目录下执行命令 — ✓ Delivered (Phase 6, raw_arg fix)
- [x] **FIX-02**: 含空格、中文、特殊字符的项目路径也能正确执行命令 — ✓ Delivered (Phase 6, raw_arg fix)

### WIN — 窗口体验

- [x] **WIN-01**: 应用窗口无边框，顶部显示自定义标题栏（应用名称 + 拖拽区域 + 最小化/最大化/关闭按钮） — ✓ Delivered (Phase 7, TitleBar component)
- [x] **WIN-02**: 无边框窗口保留窗口阴影和正常的 resize 拖拽能力 — ✓ Delivered (Phase 7, shadow + resize)
- [x] **WIN-03**: 高 DPI 显示器下无边框窗口元素和文字显示正常 — ✓ Delivered (Phase 7, validated)

### PROJ — 项目信息增强

- [x] **PROJ-07**: 用户可在项目设置图标模态中，从项目目录自动识别应用图标 — ✓ Delivered (Phase 8, scan_project_icons)
- [x] **PROJ-08**: 用户可在项目设置图标模态中，选择自定义图标文件路径 — ✓ Delivered (Phase 8, file picker + file: prefix)
- [x] **PROJ-09**: 选中项目后，指令卡片上方信息栏显示文件夹大小 — ✓ Delivered (Phase 8, get_project_info)
- [x] **PROJ-10**: Git 仓库显示当前分支名 — ✓ Delivered (Phase 8, .git/HEAD reading)
- [x] **PROJ-11**: 用户可通过"打开文件夹"按钮打开项目目录 — ✓ Delivered (Phase 9, open_folder command)

### UI — UI 优化

- [x] **UI-09**: 模态窗自适应大小，窗口过小时可滚动 — ✓ Delivered (Phase 8, flex-col + max-h-[90vh])
- [x] **UI-10**: 全局指令/项目指令切换改为按钮样式 — ✓ Delivered (Phase 9, Toggle Group)

### PRE — 预设指令系统

- [x] **PRE-01**: 添加指令模态窗包含双下拉框（分类+命令） — ✓ Delivered (Phase 10, dual Select)
- [x] **PRE-02**: 默认卡片精简为 git pull + claude — ✓ Delivered (Phase 10-01)
- [x] **PRE-03**: 预设命令库涵盖常用命令 — ✓ Delivered (Phase 10-01, 4 分类 25 命令)
- [x] **PRE-04**: 用户通过预设添加指令时，可选择全局或项目指令 — ✓ Delivered (Phase 10-03, scope selector)
