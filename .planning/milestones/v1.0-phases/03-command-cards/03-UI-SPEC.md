### Component Interaction States (Phase 3 Update)

**CommandCard** — extended with execution flash state:

| State | Visual | Duration | Trigger |
|-------|--------|----------|---------|
| Default | bg-white/5, border-white/10 | -- | Rest |
| Hover | bg-white/10, border-white/20, scale(1.02) | 150ms ease-out | Mouse enter (only when not disabled, not flashing) |
| Active | bg-white/15, scale(0.98) | 100ms ease-in | Mouse down (only when not disabled, not flashing) |
| Disabled | opacity-40, cursor-not-allowed | -- | No project selected |
| **Flashing** | **animate-card-flash (border glow + scale bounce)** | **400ms ease-out** | **Click to execute (per D-05, D-06)** |
| Focus | ring-2 ring-ring | -- | Keyboard focus |

**State mutual exclusion rules:**
- Flashing overrides hover/active: when `flashing === true`, hover and active transitions are suppressed
- Flashing prevents re-click: when `flashing === true`, click handler returns early (per RESEARCH.md Pitfall 2)
- Disabled overrides all: when `disabled === true`, no interaction states apply

## Layout Specification

### Grid Specification (Phase 3 Change)

**Before (Phase 1):** `grid grid-cols-2 gap-3` — fixed 2 columns

**After (Phase 3):** `grid grid-cols-[repeat(auto-fill,_minmax(140px,_1fr))] gap-3` — adaptive columns

| Property | Value | Rationale |
|----------|-------|-----------|
| Grid strategy | CSS Grid `auto-fill` + `minmax()` | Pure CSS responsive, zero JS (per D-08) |
| Min card width | 140px | 296px available / 2 cols - 6px gap = 142px; 140px guarantees 2 cols at min window (per D-09, RESEARCH.md Pitfall 1) |
| Max card width | 1fr (fills available) | Cards stretch to fill column, consistent sizing |
| Gap | 12px (gap-3) | Unchanged from Phase 1 |
| fill mode | `auto-fill` (not `auto-fit`) | Preserves empty columns for stable layout (per RESEARCH.md Pattern 1) |

### Column Behavior by Window Width

| Window Width | Main Area Width (approx) | Columns | Notes |
|-------------|--------------------------|---------|-------|
| 600px (min) | ~296px | 2 | D-09: minimum 2 columns guaranteed |
| 720px (default) | ~416px | 2-3 | Default window, comfortable layout |
| 900px+ | ~596px+ | 3-4 | Wide window, cards remain compact |

## Animation Specification

**Inherits Phase 1 UI-SPEC base animations.** Phase 3 adds the execution flash animation.

### Phase 1 Base Animations (unchanged)

| Element | Transition | Duration | Easing |
|---------|-----------|----------|--------|
| Command card hover | scale(1.02), bg opacity change | 150ms | ease-out |
| Command card active | scale(0.98) | 100ms | ease-in |
| Toast enter | slide-in from bottom-right | 200ms | spring |
| Toast exit | fade-out | 150ms | ease-in |

### Phase 3 New Animation: Execution Flash

| Property | Value | Rationale |
|----------|-------|-----------|
| Name | `card-flash` | Descriptive, unique in project namespace |
| Duration | 400ms | Perceptible feedback without feeling slow (per D-05) |
| Easing | ease-out | Fast start, smooth settle |
| Keyframe sequence | 0%: border glow + scale(0.97) press -> 50%: peak glow -> 100%: rest | Simulates "press + release" physical feedback |
| Icon behavior | `animate-spin` on icon during flash | Tailwind built-in, zero extra code; signals "executing" |
| Recovery | `setTimeout(() => setFlashing(false), 420)` | 400ms animation + 20ms buffer (per RESEARCH.md Pattern 2) |

### @keyframes Definition

```css
/* src/index.css — inside @theme inline block */
--animate-card-flash: card-flash 400ms ease-out;

/* src/index.css — OUTSIDE @theme inline block, same file */
@keyframes card-flash {
  0% {
    border-color: rgba(255, 255, 255, 0.3);
    box-shadow: 0 0 12px rgba(255, 255, 255, 0.1);
    transform: scale(0.97);
  }
  50% {
    border-color: rgba(255, 255, 255, 0.5);
    box-shadow: 0 0 20px rgba(255, 255, 255, 0.15);
  }
  100% {
    border-color: rgba(255, 255, 255, 0.1);
    box-shadow: none;
    transform: scale(1);
  }
}
```

### Animation Hierarchy

| Priority | State | Behavior |
|----------|-------|----------|
| 1 (highest) | Flashing | animate-card-flash overrides all; hover/active suppressed |
| 2 | Active | scale(0.98) press feedback |
| 3 | Hover | scale(1.02) + bg brightness increase |
| 4 | Default | No animation, static rest state |

## Interaction Flow

### Card Click Execution Flow

```
User clicks card
    |
    v
[flashing state check] --flashing?--> IGNORE (prevent double-click)
    |
    v (not flashing)
setFlashing(true)
    |
    +---> animate-card-flash plays (400ms)
    +---> icon animate-spin plays (400ms)
    +---> onClick() fires --> onExecute(cmd) --> invoke Rust backend
    |
    v
setTimeout(420ms) --setFlashing(false)--> Rest state
    |
    v
Toast notification appears ("已执行: npm run build")
```

### Empty State Flow (per D-01, D-02)

```
No project selected
    |
    v
MainArea renders empty state guide
    |
    +---> FolderOpen icon (size-12, text-muted-foreground)
    +---> "选择一个项目开始" (text-lg font-semibold)
    +---> "从左侧添加或选择项目，然后点击指令卡片执行" (text-sm text-muted-foreground)
    |
    v
No command cards rendered (CMD-08 satisfied by absence)
```

## Accessibility

**Inherits Phase 1 UI-SPEC accessibility baseline.** Phase 3 additions:

| Requirement | Implementation |
|-------------|---------------|
| Card aria-label | Unchanged: aria-label={name} on each CommandCard |
| Execution feedback | Visual only (card-flash animation). No aria-live announcement needed -- Toast (sonner) already provides accessible announcement of execution result |
| Tooltip (title) | Native HTML title attribute provides accessible name supplement |
| Flash animation | Respect `prefers-reduced-motion`: add `motion-reduce:animate-none` to card container (suppress animation for users who prefer reduced motion) |

### Reduced Motion Support

```tsx
// In CommandCard className, add:
```
