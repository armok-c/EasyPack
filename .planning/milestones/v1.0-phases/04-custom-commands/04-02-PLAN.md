**新增计算属性 — commands（当前应显示的指令列表）：**
```typescript
const commands = useMemo(() => {
  // 全局模式：预设 + 全局自定义，按 addedAt 排序
  return [...getPresetAsCommandItems(), ...customCommands].sort((a, b) => a.addedAt - b.addedAt);
}, [customCommands]);
```

注意：项目级覆盖逻辑在 Plan 03 中实现，此处 commands 只处理全局模式。

**新增操作函数：**

1. `addCommand(name: string, command: string, icon?: string)` — 添加全局自定义指令
   - 创建新 CommandItem: `{ id: crypto.randomUUID(), name, command, icon: icon ?? DEFAULT_ICON, type: 'custom', scope: 'global', addedAt: Date.now() }`
   - 追加到 customCommands 数组
   - `await store.set(CUSTOM_COMMANDS_KEY, updated)`
   - `toast.success("已添加指令: {name}")`

2. `updateCommand(id: string, data: { name: string; command: string; icon: string })` — 更新指令
   - 在 customCommands 中找到 id 对应的项
   - 用 immutable update 更新 name/command/icon
   - `await store.set(CUSTOM_COMMANDS_KEY, updated)`
   - `toast.success("已保存指令: {name}")`

3. `deleteCommand(id: string)` — 删除指令
   - 从 customCommands 中过滤掉 id 对应的项
   - `await store.set(CUSTOM_COMMANDS_KEY, updated)`
   - `toast.success("已删除指令: {name}")`

4. 扩展 `removeProject` — 删除项目时清理项目级指令数据
   - 在现有 removeProject 逻辑中增加：`await store?.delete(projectCommandsKey(id))`

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | I (Information Disclosure) | Store JSON 文件 | accept | tauri-plugin-store 默认存储在应用数据目录，仅当前用户可访问 |
