### Sidebar Project Item Component Detail

```tsx
// Single project item in sidebar list
// Parent provides: project, isSelected, onSelect, onRemove
<div className={cn(
  "group relative flex items-center px-2 py-2 rounded-lg border cursor-pointer",
  "transition-all duration-150",
  isSelected
    ? "bg-white/10 border-white/20"                        // D-06
    : "bg-white/5 border-white/10 hover:bg-white/[0.08]", // D-07, D-08
)}>
  {/* Project name - truncate if too long (D-03: folder name only) */}
  <span className="text-xs text-foreground truncate flex-1">
    {project.name}
  </span>

  {/* Delete button - visible only on hover (D-09) */}
  <button
    onClick={(e) => { e.stopPropagation(); onRemove(project.id); }}
    className="opacity-0 group-hover:opacity-100 ml-1 p-0.5 rounded
               hover:bg-white/10 transition-opacity duration-150"
    aria-label={`删除项目 ${project.name}`}
  >
    <X className="size-3 text-muted-foreground" />
  </button>
</div>
```

## Data Persistence Specification

### Store Configuration

| Property | Value | Source |
|----------|-------|--------|
| Plugin | tauri-plugin-store 2.4.2 | RESEARCH.md |
| Store filename | easypack-store.json | RESEARCH.md |
| autoSave | 100ms (default debounce) | D-12, RESEARCH.md |
| Store location | app_data_dir (Tauri managed) | RESEARCH.md |

### Data Schema

```typescript
// Key: "projects" — 项目列表数组
interface ProjectItem {
  id: string;       // 规范化路径 (lowercase, forward slashes)
  name: string;     // 文件夹名称
  path: string;     // 原始完整路径 (preserves original casing)
  addedAt: number;  // Date.now() timestamp
}

// Key: "selectedProjectId" — 当前选中项目 ID
type SelectedProjectId = string | null;

// Project ID generation (RESEARCH.md Pattern 2)
function generateProjectId(path: string): string {
  return path
    .split(/[\\/]/)          // handle both / and \
    .filter(Boolean)
    .join('/')
    .toLowerCase();          // Windows case-insensitive
}
```

### Store Operations

| Operation | Store Key | React State | Store Method |
|-----------|-----------|-------------|-------------|
| Init load | projects, selectedProjectId | setProjects, setSelectedId | store.get() |
| Add project | projects, selectedProjectId | append to projects, set selectedId | store.set() x2 |
| Remove project | projects, selectedProjectId | filter projects, recalc selectedId (D-10) | store.set() x2 |
| Select project | selectedProjectId | setSelectedId | store.set() |

### Loading State

| State | Visual Treatment |
|-------|-----------------|
| Loading (store init) | No visual change — keep Phase 1 empty state, store loads in <100ms |
| Loaded with data | Show project list with restored selection |
| Loaded without data | Show empty state ("还没有项目") |
| Store load failure | Graceful degradation to memory-only mode (console.warn, no error toast) |

## Interaction Flow Specification

### Add Project Flow

1. User clicks "添加项目" button
2. System folder picker opens (dialog title: "选择项目文件夹")
3. User selects folder -> name extracted from path
4. Duplicate check via normalized path ID (D-04)
   - If duplicate: toast.error("项目已存在"), no addition
   - If new: add to projects array, auto-select new project
5. Project appears at bottom of list (D-02), selected state applied (D-06)

### Select Project Flow

1. User clicks on project item in sidebar
2. selectedId updates to clicked project ID
3. Visual state changes to selected (bg-white/10, border-white/20) -- D-06
4. Previously selected item returns to unselected state (bg-white/5) -- D-07
5. Main area updates to show selected project's info and command cards (D-05)

### Delete Project Flow

1. User hovers over project item -> X button appears (D-09)
2. User clicks X button -> e.stopPropagation() prevents selection change
3. Project removed from array
4. Auto-select nearest neighbor (D-10):
   - If deleted project was selected: select next item, or previous if no next
   - If deleted project was not selected: keep current selection
5. If last project deleted: show empty state (D-11)
