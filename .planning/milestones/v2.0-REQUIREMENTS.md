# Milestone v2.0 Requirements Archive: 能力跃升

## Multi-line Script Commands (SCRIPT)

- [x] **SCRIPT-01**: 用户可以在指令编辑器中编写多行命令，支持 Windows 批处理语法（if/else、for、变量、goto 等） — Phase 17
- [x] **SCRIPT-02**: 多行脚本写入临时 .bat 文件由 cmd.exe 执行，编码使用 `chcp 65001` 确保 UTF-8 兼容 — Phase 17
- [x] **SCRIPT-03**: 脚本编辑器提供语法高亮和行号显示（基于 CodeMirror 6） — Phase 17
- [x] **SCRIPT-04**: 用户可选择执行模式：严格模式（失败即停，用 `&&` 连接）或宽松模式（继续执行，用 `&` 连接） — Phase 17
- [x] **SCRIPT-05**: 现有单行指令数据模型向后兼容，`scriptLines` 为可选字段 — Phase 17

## Version Management (VER)

- [x] **VER-01**: 应用内显示当前版本号（标题栏或设置页） — Phase 16
- [x] **VER-02**: 启动时检查 GitHub Releases API 是否有新版本，结果缓存 24 小时避免速率限制 — Phase 16
- [x] **VER-03**: 发现新版本时显示更新提示（badge 或 toast 通知） — Phase 16
- [x] **VER-04**: 点击更新提示后打开浏览器跳转到 GitHub Release 下载页面 — Phase 16

## Keyboard Shortcut Panel (KBD)

- [x] **KBD-01**: 提供类似 VS Code 的快捷键设置面板，列出所有可绑定的操作 — Phase 18
- [x] **KBD-02**: 用户可在面板中点击操作后按键录制新快捷键组合 — Phase 18
- [x] **KBD-03**: 快捷键冲突检测和警告提示（复用 v1.2 已有逻辑） — Phase 18
- [x] **KBD-04**: 快捷键搜索、按分类筛选、重置为默认值功能 — Phase 18
- [x] **KBD-05**: 除指令执行外，增加窗口操作、项目切换、打开项目文件夹等可绑定操作 — Phase 18
- [x] **KBD-06**: 快捷键绑定持久化保存，重启后恢复 — Phase 18

## Floating Window Improvements (FLOAT)

- [x] **FLOAT-01**: 悬浮窗布局更紧凑，显示信息更精简 — Phase 19
- [x] **FLOAT-02**: 悬浮窗可折叠/展开，折叠态只显示项目图标和名称 — Phase 19
- [x] **FLOAT-03**: 折叠态点击项目名称可切换当前选中项目 — Phase 19
- [x] **FLOAT-04**: 折叠/展开之间有平滑动画过渡 — Phase 19
- [x] **FLOAT-05**: 悬浮窗支持拖拽移动到屏幕任意位置 — Phase 19

## Auto-start on Boot (BOOT)

- [x] **BOOT-01**: 提供开机启动开关（tauri-plugin-autostart） — Phase 15
- [x] **BOOT-02**: 开机启动后自动隐藏主窗口到系统托盘 — Phase 15
- [x] **BOOT-03**: 每次启动时验证注册表条目是否有效，无效则自动重新注册 — Phase 15
- [x] **BOOT-04**: 开机启动设置持久化，重启后保持 — Phase 15

## Multi-config Profile System (CONFIG)

- [x] **CONFIG-01**: 用户可创建、删除、重命名配置 profile — Phase 20
- [x] **CONFIG-02**: 用户可在不同 profile 之间切换，切换后加载对应数据 — Phase 20
- [x] **CONFIG-03**: 用户可导出当前配置为 JSON 文件 — Phase 20
- [x] **CONFIG-04**: 用户可导入 JSON 配置文件恢复配置 — Phase 20
- [x] **CONFIG-05**: 从现有单配置自动迁移到 profile 系统 — Phase 20
- [x] **CONFIG-06**: Profile 切换使用 mutex 序列化，防止并发写入 — Phase 20
