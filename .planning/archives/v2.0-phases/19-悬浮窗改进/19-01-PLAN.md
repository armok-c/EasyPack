# Plan 19-01: FloatApp 折叠/展开 UI + 紧凑布局

将悬浮窗从固定展开态升级为可折叠/展开的双态 UI。折叠态为 130px 宽胶囊形，仅显示项目图标+名称+展开箭头；展开态保持 220px 指令列表但更紧凑。折叠/展开通过 CSS 过渡动画平滑衔接（300ms ease-in-out）。两个状态均支持拖拽。

## Tasks

### Task 1: 添加折叠/展开状态机 + 胶囊折叠态 UI

<read_first>
- src/components/FloatApp.tsx — 当前完整的悬浮窗组件（122 行），需在 return JSX 中添加折叠态分支
- src/lib/icons.ts — getIconByName 函数，折叠态复用同一图标逻辑
- src/components/Sidebar.tsx:105-118 — 项目图标渲染模式（LucideIcon + file icon + fallback），折叠态复用
</read_first>

<action>
在 FloatApp.tsx 中添加折叠/展开状态机：

1. 添加 state: `const [collapsed, setCollapsed] = useState(false)` — 默认展开态
2. 添加 `LogicalSize` 的 import from `@tauri-apps/api/dpi`（用于后续 resize，但此 task 先只用 CSS 处理视觉）
3. 添加 `ChevronDown` 和 `ChevronRight` 的 import from `lucide-react`（折叠态展开箭头指示器和展开态折叠指示器）
4. 添加 `isFileIcon` 和 `getFilePath` 的 import from `@/lib/icons`
5. 添加 `convertFileSrc` 的 import from `@tauri-apps/api/core`

改造 return JSX 为条件分支：

**折叠态（collapsed === true）：**
- 根 div 改为 `w-[130px]` + `rounded-full`（胶囊形，per D-02/D-03）
- 高度使用固定值 `h-[32px]`
- 内部为水平布局：左项目图标 + 中项目名称 + 右展开箭头（ChevronRight）
- 项目图标区域（14px）：复用 Sidebar 的图标渲染逻辑（getIconByName 或 file icon 或首字母圆圈 fallback）
  - 当 project.icon 存在且为 file icon 时，渲染 img 标签 size-3.5
  - 当 project.icon 存在且非 file icon 时，渲染 LucideIcon 组件 size-3.5
  - 当 project.icon 不存在时，渲染首字母圆圈：一个 `size-3.5 rounded-full bg-muted flex items-center justify-center text-[8px] font-medium text-muted-foreground` 的 div，内含 `project.name.charAt(0).toUpperCase()`
- 项目名称：`text-[11px]`（per D-04），`truncate max-w-[60px]`
- 展开箭头（ChevronRight）：`size-3 text-muted-foreground`，点击时 `setCollapsed(false)`
- 整个胶囊体的 mousedown 绑定 handleDragStart（保持拖拽能力，per D-05/FLOAT-05）
- 展开箭头按钮的 click 事件需要 `e.stopPropagation()` 防止触发拖拽
- 无项目时显示 "EasyPack" 文字 + FolderOpen 图标，整体不可点击切换项目

**展开态（collapsed === false）— 保持现有结构但稍作调整：**
- 根 div 保持 `w-[220px]`
- Header 区域：左侧项目名称 + 右侧折叠按钮（ChevronDown 图标，替代原来的 X 关闭按钮位置）+ 关闭按钮（X 图标）
  - 折叠按钮点击时调用 `setCollapsed(true)`
  - Header 空白区域点击也触发 `setCollapsed(true)`（per D-09），但排除按钮点击
  - 保持关闭按钮的 handleClose 功能
- 内容区域保持现有指令列表渲染不变

**状态切换处理：**
- 折叠态 → 展开态：点击 ChevronRight 按钮时 setCollapsed(false)
- 展开态 → 折叠态：点击 ChevronDown 按钮或 header 空白区域时 setCollapsed(true)
- 确保拖拽（handleDragStart）在两个状态下均正常工作
</action>

### Task 2: CSS 过渡动画 + 展开态紧凑布局

<read_first>
- src/components/FloatApp.tsx — Task 1 完成后的文件，包含折叠/展开条件分支
</read_first>

<action>
在 FloatApp.tsx 中添加 CSS 过渡动画和紧凑布局：

1. **过渡动画（per D-15/D-16/D-17/D-18）：**
   - 在根 div 上添加 CSS transition：使用 `transition-all duration-300 ease-in-out`
   - 折叠态根 div 的 transition 确保 width、border-radius 等属性平滑变化
   - 注意：实际窗口物理 resize 由 Plan 02 的 useFloatWindow 处理（通过 Tauri setSize），此 task 仅处理视觉过渡的 CSS 部分

2. **展开态紧凑化（per FLOAT-01, D-07）：**
   - Header 高度从 `h-[28px]` → `h-[24px]`
   - Header 内边距从 `px-2` → `px-1.5`
   - 内容区域内边距从 `p-2` → `px-1.5 py-1`
   - 指令行高度从 `h-8` → `h-7`
   - 指令行内边距从 `px-2` → `px-1.5`
   - 字体保持 text-xs（已经是最小合理尺寸）
   - 项目名称字体保持 text-xs
   - 关闭按钮和折叠按钮尺寸从 `size-4` → `size-3.5`，图标从 `size-3` → `size-2.5`
   - 指令行图标从 `size-[14px]` → `size-[12px]`

3. **折叠态样式微调：**
   - 胶囊内各元素间距使用 `gap-1`
   - 折叠态背景风格与展开态统一（`bg-background`，per D-05）
   - 折叠态边框使用 `border border-white/10`（与展开态一致）

4. **确保拖拽在过渡期间不中断：**
   - handleDragStart 函数无需修改（已排除 button 元素）
   - 折叠态中展开箭头为 button 元素，会被 `target.closest("button")` 排除，不影响拖拽
</action>
