Per D-01: 使用 `std::os::windows::process::CommandExt::raw_arg()` 替换 `.args([...])` 调用。

**具体修改 execute_command 函数：**

1. 在文件顶部添加 import：
```rust
use std::os::windows::process::CommandExt;
```

2. 保留 `build_full_command()` 函数不变（它生成的 `cd /d "path" && command` 格式字符串是正确的）。

3. 替换 WT 路径（当前第 18-20 行）：
```rust
// 当前（BUG）:
StdCommand::new("wt")
    .args(["new-tab", "cmd", "/K", &full_command])
    .spawn();

// 修改为（per D-01）:
let escaped_command = full_command.replace('"', "\"\"");
StdCommand::new("wt")
    .raw_arg(format!("new-tab cmd /K \"{}\"", escaped_command))
    .spawn();
```

注意：`raw_arg` 传递的字符串中，`full_command` 已经包含引号包裹的路径。WT 的 `/K` 参数后面需要把整个命令字符串作为一个参数传给 cmd。外层需要再包一层引号。cmd.exe 的引号转义规则是 `""`。最终 `raw_arg` 内容为：
```
new-tab cmd /K "cd /d ""path"" && command"
```

4. 替换 cmd 回退路径（当前第 27-29 行）：
```rust
// 当前（BUG）:
StdCommand::new("cmd")
    .args(["/C", "start", "cmd", "/K", &full_command])
    .spawn()

// 修改为（per D-01）:
let escaped_command = full_command.replace('"', "\"\"");
StdCommand::new("cmd")
    .raw_arg(format!("/C start cmd /K \"{}\"", escaped_command))
    .spawn()
```

5. 保留所有 4 个现有单元测试（test_build_full_command_basic, test_build_full_command_path_with_spaces, test_build_full_command_path_with_cjk, test_build_full_command_all_preset_commands），它们测试 build_full_command 函数，该函数不需要修改。

6. 新增一个测试验证引号转义逻辑：
```rust
#[test]
fn test_raw_arg_escaping() {
    // 验证含空格路径的引号转义
    let full_cmd = build_full_command("D:\\My Projects\\app", "npm run dev");
    let escaped = full_cmd.replace('"', "\"\"");
    // 期望: cd /d ""D:\My Projects\app"" && npm run dev
    assert!(escaped.contains("\"\"D:\\My Projects\\app\"\""));
}
```
