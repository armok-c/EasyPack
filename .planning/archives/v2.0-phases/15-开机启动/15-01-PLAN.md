    **1. Cargo.toml:** 在 [dependencies] 中追加 `tauri-plugin-autostart = "2"` (与其他 tauri-plugin 依赖排列在一起)。

    **2. capabilities/default.json:** 在 permissions 数组末尾追加三个 autostart 权限: "autostart:allow-enable", "autostart:allow-disable", "autostart:allow-is-enabled"。

    **3. lib.rs — 插件注册 (per D-01):** 在 tauri::Builder::default() 的 .plugin() 链中追加 tauri_plugin_autostart 插件注册，使用 Builder API 模式:
    - 使用 `tauri_plugin_autostart::Builder::new().arg("--autostart").build()` 构建插件
    - 放在 `.plugin(tauri_plugin_single_instance::init(...))` 之后、`.plugin(tauri_plugin_dialog::init())` 之前（或之后，与其他 plugin 排列自然即可）

    **4. lib.rs — setup 闭包中添加 --autostart 检测与窗口隐藏 (per D-03, D-04):**
    在 setup 闭包的最开始（全局菜单事件处理器之前）插入:
    - 使用 `std::env::args().any(|arg| arg == "--autostart")` 检测命令行参数
    - 如果检测到 --autostart:
      a. 获取 main webview window 并调用 `window.hide()` (WebView 加载前隐藏，避免闪现)
      b. 调用 `window.set_skip_taskbar(true)` 避免在任务栏出现
      c. 通过 `app.emit("app:autostart-hidden", ())` 通知前端 (per D-04)
      d. D-02 自启防御: 读取 plugin-store 中 "trayEnabled" 值，如果为 false 或不存在，静默设为 true 并写入 store，确保开机自启后应用有托盘可去。同时也检查 "closeToTray" 值，如果为 false 则同样静默设为 true
    - 注意: `use tauri::Emitter;` 和 `use tauri::Manager;` 导入放在 setup 闭包内部（与现有模式一致）

    **5. lib.rs — setup 闭包中添加自愈逻辑 (per D-05, D-06, D-07, D-08):**
    在 --autostart 检测逻辑之后、全局菜单事件处理器之前插入:
    - 使用 `use tauri_plugin_autostart::ManagerExt;` 获取 autolaunch manager
    - 读取 plugin-store 中的 autostartEnabled 值: 使用 `tauri_plugin_store::StoreExt` 获取 store，读取 "autostartEnabled" 键
    - 如果 autostartEnabled 为 true，调用 `autolaunch_manager.is_enabled()` 检查注册表
    - 如果 is_enabled() 返回 false (或 Err)，静默调用 `autolaunch_manager.enable()` 重新注册
    - 所有错误静默处理 (per D-07)，用 `let _ = ...` 或 `if let Ok(...) = ...` 模式
    - 自愈逻辑必须在实际的 Rust setup 中执行，不委托给前端 (per D-08)
