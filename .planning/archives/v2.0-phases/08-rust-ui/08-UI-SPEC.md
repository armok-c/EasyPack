## Interaction Specifications

### 1. Dialog Self-Adaptive Height (UI-09, per D-09, D-10)

#### Problem

When the window is small, DialogContent content gets clipped with no way to scroll.

#### Solution

Modify DialogContent base styles to add max-height constraint and internal scrolling.

#### DialogContent Layout

```
+--[DialogContent max-h-[90vh] flex flex-col]--+
|                                                |
|  [DialogHeader — fixed, does not scroll]       |
|                                                |
|  +--[Content area: flex-1 overflow-y-auto]--+  |
|  |                                            |
|  |  (scrollable content)                      |
|  |                                            |
|  +--------------------------------------------+
|                                                |
|  [DialogFooter — fixed, does not scroll]       |
|                                                |
+------------------------------------------------+
```

#### Implementation

DialogContent base className changes:
- Add `max-h-[90vh]` to outer container
- Change from `grid` to `flex flex-col` layout
- DialogHeader: remains at top, no scroll
- Middle content: `flex-1 overflow-y-auto min-h-0`
- DialogFooter: remains at bottom, no scroll

The `min-h-0` on the content area is critical -- it allows the flex child to shrink below its content size, enabling overflow scrolling.

#### Affected Dialogs

All dialogs using DialogContent inherit the fix automatically:
- ProjectSettingsDialog
- CommandDialog
- Any future dialogs

### 2. Icon Auto-Detection (PROJ-07, per D-01, D-02, D-03)

#### Flow

```
User right-clicks project -> "设置图标和颜色" -> ProjectSettingsDialog opens
User sees existing icon grid (lucide icons) + new section below:
  "自定义图标"
  [从项目导入] [选择文件]

User clicks "从项目导入":
  -> invoke("scan_project_icons", { path: project.path })
  -> Loading spinner replaces candidate area
  -> Rust scans: package.json icon, Cargo.toml icon, favicon.ico, app-icon.png, icon.png, logo.png, etc.
  -> Returns candidate file paths
  -> If candidates found: show thumbnail grid (32px icons, filename below)
  -> If no candidates: show "未找到可用图标"
  -> If error: show "图标扫描失败，请重试"

User clicks candidate icon:
  -> Selected icon highlighted (ring-1 ring-primary/50)
  -> Preview updates with file icon
  -> Save button becomes active

User clicks "选择文件":
  -> Tauri dialog file picker opens (filter: .ico, .png, .svg)
  -> User selects file
  -> Preview updates with selected file
  -> Save button becomes active

User clicks "保存设置":
  -> onSave fires with { icon: "file:/path/to/icon.png" } or { icon: "lucide:Terminal" }
  -> Sidebar re-renders project with appropriate icon renderer
```

#### Icon Type Discrimination

The ProjectItem.icon field needs to support two types:
- Lucide icon name: stored as-is, e.g., "Terminal", "Code"
- File path icon: prefixed with "file:", e.g., "file:C:/Projects/app/icon.png"

This prefix convention allows Sidebar and MainArea to discriminate rendering without changing the field type.

#### Candidate Icon Grid Layout

```
+----------------------------------------------+
|  自定义图标                                    |
|  [从项目导入]  [选择文件]                      |
|                                               |
|  +--------+  +--------+  +--------+          |
|  | [img]  |  | [img]  |  | [img]  |          |
|  | icon.. |  | logo.. |  | favic..|          |
|  +--------+  +--------+  +--------+          |
+----------------------------------------------+
```

Each candidate:
- Thumbnail: 32px (size-8), rounded-md, border border-white/10, bg-white/5
- Filename: text-xs truncate max-w-[32px], text-muted-foreground
- Clickable with same selection pattern as lucide icon grid

#### Candidate Icon Grid Behavior

| State | Style |
|-------|-------|
| Default | bg-white/5, border-white/10, rounded-md |
| Hover | bg-white/10, border-white/20 |
| Selected | bg-white/15, border-primary, ring-1 ring-primary/50 |
| Loading | Skeleton pulse animation on placeholder rectangles |
| Empty | "未找到可用图标" text with FolderOpen icon |

### 3. Sidebar File Icon Rendering (D-02)

#### Rendering Logic

```typescript
// Pseudocode for icon rendering in SortableProjectItem
if (project.icon?.startsWith("file:")) {
  const filePath = project.icon.slice(5); // strip "file:" prefix
  return (
    <img
      src={convertFileSrc(filePath)}  // Tauri asset protocol
      alt=""
      className="size-3.5 mr-1.5 flex-shrink-0 rounded-sm object-cover"
      onError={(e) => {
        // Fallback: replace with default Folder icon
        e.currentTarget.style.display = "none";
      }}
    />
  );
} else if (project.icon) {
  const ProjectIcon = getIconByName(project.icon);
  return <ProjectIcon className="size-3.5 mr-1.5 flex-shrink-0 text-muted-foreground" />;
}
```

The `convertFileSrc` function from `@tauri-apps/api/core` converts local file paths to Tauri asset protocol URLs that the WebView can render.

#### Fallback Chain

1. Custom file icon (if `icon` starts with "file:") -- try to render
2. On image load error -> hide image element, show nothing (no icon)
3. Lucide icon (if `icon` is a lucide name) -- render component
4. No icon field -> no icon displayed (existing behavior)

### 4. Folder Size & Git Branch Display (PROJ-09, PROJ-10, per D-04 through D-08)

#### Info Bar Layout (MainArea.tsx, lines 162-205)

Current layout:
```
当前项目: {name}                     [编辑按钮]
{path}
{mode label} · {mode switch link}
```

New layout:
```
当前项目: {name}                     [编辑按钮]
{path}
{size text} · {branch text}
{mode label} · {mode switch link}
```

#### Info Display Rules

| Data | Available | Unavailable |
|------|-----------|-------------|
| Folder size | "{size}" (e.g., "12.3 MB") | "计算中..." (during calc) or "无法计算" (on error) |
| Git branch | "分支: {name}" (e.g., "分支: main") | Entire branch text hidden (no "未找到分支" text) |

#### Folder Size Formatting

Human-readable format, calculated in Rust backend:
- < 1 KB: "xxx B"
- < 1 MB: "xxx KB" (1 decimal: "12.3 KB")
- < 1 GB: "xxx MB" (1 decimal: "123.4 MB")
- >= 1 GB: "xxx GB" (1 decimal: "1.2 GB")

#### Fetch Timing (per D-04)

- Triggered when user selects a project (call invoke on project switch)
- Display "计算中..." while waiting
- Timeout: 8 seconds (Claude's discretion, between 5-10s from D-06)
- On timeout: show "无法计算"
- Result cached in component state, re-fetched on project switch

#### Combined Rust Command (per D-08)

Single `invoke("get_project_info", { path })` returns:
```rust
struct ProjectInfo {
    size: String,        // Human-readable, e.g., "12.3 MB"
    branch: Option<String>,  // None if not a git repo or detached HEAD
}
```
