    将现有 className 从:
    ```
    "fixed top-[50%] left-[50%] z-50 grid w-full max-w-[calc(100%-2rem)] translate-x-[-50%] translate-y-[-50%] gap-4 rounded-lg border bg-background p-6 shadow-lg duration-200 outline-none data-[state=closed]:animate-out data-[state=closed]:fade-out-0 data-[state=closed]:zoom-out-95 data-[state=open]:animate-in data-[state=open]:fade-in-0 data-[state=open]:zoom-in-95 sm:max-w-lg"
    ```

    改为:
    ```
    "fixed top-[50%] left-[50%] z-50 flex flex-col w-full max-w-[calc(100%-2rem)] max-h-[90vh] translate-x-[-50%] translate-y-[-50%] rounded-lg border bg-background p-6 shadow-lg duration-200 outline-none data-[state=closed]:animate-out data-[state=closed]:fade-out-0 data-[state=closed]:zoom-out-95 data-[state=open]:animate-in data-[state=open]:fade-in-0 data-[state=open]:zoom-in-95 sm:max-w-lg"
    ```

    变更点（共 3 处）:
    1. `grid` -> `flex flex-col` (per D-10: flex 布局支持固定 header/footer + 滚动内容区)
    2. 添加 `max-h-[90vh]` (per D-09: 全局最大高度限制，90vh 来自 UI-SPEC 的决定)
    3. 移除 `gap-4` (flex 布局下间距由内部结构控制，gap-4 会导致滚动包装区域与 header/footer 之间有不一致的间距)

    **变更 2: 修改 DialogContent 内部结构**

    DialogContent 当前内部结构（简化）:
    ```tsx
    <DialogContentOverlay>
      <DialogContentPrimitive>
        {children}  <!-- DialogHeader, 中间内容, DialogFooter 混在一起 -->
        <CloseButton />
      </DialogContentPrimitive>
    </DialogContentOverlay>
    ```

    需要改为:
    ```tsx
    <DialogContentOverlay>
      <DialogContentPrimitive className="...flex flex-col max-h-[90vh]...">
        {children}  <!-- 保持 children 传递不变 -->
        <CloseButton />
      </DialogContentPrimitive>
    </DialogContentOverlay>
    ```

    **关键：不改变 DialogContent 的 children 传递方式。** DialogHeader、中间内容、DialogFooter 作为 children 传入，DialogContent 本身不拆分它们。但为了让中间内容区域能滚动，需要在 DialogContent 内部添加一个滚动包装。

    **实现方案：使用 React.Children 自动分离和包装**

    DialogContent 内部需要自动将 children 分为三部分：
    1. DialogHeader 类型的子元素 -> 渲染在滚动区外（固定在顶部）
    2. DialogFooter 类型的子元素 -> 渲染在滚动区外（固定在底部）
    3. 其他子元素（中间内容）-> 包装在 `flex-1 overflow-y-auto min-h-0` 的滚动容器中

    实现方式：使用 React.Children.map 遍历 children，检查每个 child 的 type displayName 或 type 本身来区分 DialogHeader / DialogFooter / 其他内容。对于无法识别类型的情况（如函数子组件），将所有非 Header/Footer 的内容放入滚动容器。

    具体实现:
    ```tsx
    function DialogContent({
      className,
      children,
      ...props
    }: React.ComponentPropsWithoutRef<typeof DialogPrimitive.Content>) {
      // 分离 header、footer 和中间内容
      let headers: React.ReactNode[] = [];
      let footers: React.ReactNode[] = [];
      let content: React.ReactNode[] = [];

      React.Children.forEach(children, (child) => {
        if (React.isValidElement(child)) {
          // 检查是否为 DialogHeader 或 DialogFooter
          const type = child.type;
          const displayName =
            typeof type === "function" || typeof type === "object"
              ? (type as any).displayName
              : null;

          if (displayName === "DialogHeader" || type === DialogHeader) {
            headers.push(child);
          } else if (displayName === "DialogFooter" || type === DialogFooter) {
            footers.push(child);
          } else if (child.type === DialogClose) {
            // CloseButton 保持原位
            content.push(child);
          } else {
            content.push(child);
          }
        } else {
          content.push(child);
        }
      });

      return (
        <DialogPortal>
          <DialogOverlay />
          <DialogPrimitive.Content
            className={cn(
              "fixed top-[50%] left-[50%] z-50 flex flex-col w-full max-w-[calc(100%-2rem)] max-h-[90vh] translate-x-[-50%] translate-y-[-50%] rounded-lg border bg-background p-6 shadow-lg duration-200 outline-none data-[state=closed]:animate-out data-[state=closed]:fade-out-0 data-[state=closed]:zoom-out-95 data-[state=open]:animate-in data-[state=open]:fade-in-0 data-[state=open]:zoom-in-95 sm:max-w-lg",
              className
            )}
            {...props}
          >
            {/* DialogHeader: 固定在顶部，不随内容滚动 */}
            {headers}
            {/* 中间内容: 可滚动区域 (per D-10) */}
            <div className="flex-1 overflow-y-auto min-h-0">
              {content}
            </div>
            {/* DialogFooter: 固定在底部，不随内容滚动 */}
            {footers}
          </DialogPrimitive.Content>
        </DialogPortal>
      );
    }
    ```

    **注意：** 上述实现中的 DialogClose 判断需要保留，因为现有的 CloseButton（X 按钮）是 DialogPrimitive.Close 的包装，它应该在滚动区外。如果 CloseButton 不在 children 中而是单独渲染的，则不需要特别处理。

    **检查现有 dialog.tsx 的实际实现结构：** 仔细读取 dialog.tsx 确认 CloseButton 的位置和 children 的传递方式，确保修改与现有结构兼容。

    **如果现有结构中 CloseButton 是在 children 之外单独渲染的（不是 children 的一部分）：**
    那么只需分离 DialogHeader 和 DialogFooter，其他 children 全部放入滚动容器。CloseButton 保持原位。

    **如果现有结构中 CloseButton 是 children 的一部分：**
    需要将 DialogPrimitive.Close 也识别为特殊元素，放在滚动容器外。

    **关于 min-h-0 (per RESEARCH Pitfall 2):** min-h-0 是必须的。CSS flexbox 中子元素默认 min-height: auto（等于内容高度），会阻止子元素缩小。min-h-0 允许滚动容器在空间不足时缩小并触发 overflow 滚动。

    **关于关闭按钮定位:**
    现有关闭按钮使用 `absolute top-4 right-4`，改为 flex-col 后 absolute 定位仍然有效（DialogContent 是 positioned element，flex 容器默认是 positioned context），不需要修改。

    **验证现有组件兼容性:**
    - ProjectSettingsDialog: 使用 DialogContent -> DialogHeader -> div.py-4 -> DialogFooter 结构。修改后 header 在滚动区外固定，div.py-4 内容在滚动区内，footer 在滚动区外固定。完美匹配 D-10 要求。
    - 其他使用 DialogContent 的组件同理，自动继承滚动行为。

    **注意事项:**
    - DialogHeader 和 DialogFooter 组件需要添加 displayName 静态属性以便识别。检查是否已有 displayName，如果没有则添加。
    - 如果使用 displayName 方式不可靠（生产构建可能移除 displayName），改用更可靠的方式：导出独立的 symbol 或使用 context。但鉴于本项目是个人工具、不做代码混淆，displayName 方式足够可靠。
    - 另一种更简单的方案：不自动分离 children，而是让 DialogContent 只做 max-h-[90vh] + flex flex-col 的外层容器，然后让各使用 DialogContent 的组件自己用 `DialogScrollContent` 包装滚动区域。但这违反 D-10 "使所有使用 DialogContent 的模态窗都具备自适应能力" 的要求，各组件不应需要自行添加 overflow 处理。因此必须采用自动分离方案。
