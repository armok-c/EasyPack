## Interaction Specifications

### 1. Shortcut Badge on CommandCard (KB-01, D-03, D-04, D-05)

#### Badge Position and Layout

The shortcut badge is positioned at `absolute top-1 left-1` on the CommandCard, same position as the current `shortcutNumber` badge. It replaces the number badge entirely for cards that have a custom shortcut assigned.

```
+--[CommandCard]----------------------------------+
| [Ctrl+G]                           [X] (edit)  |
|                                                  |
|              [Icon: size-6]                      |
|               指令名称                            |
+--------------------------------------------------+
```

When both a number key shortcut (1-9) and a custom shortcut exist for the same card, the custom shortcut badge takes priority in display. The number key still works via DOM-level listener as a secondary shortcut.

### 2. Recording Mode Interaction (KB-01, D-03, D-06, D-07)

#### How Recording Works

1. User enters edit mode (clicks Settings gear icon)
2. Cards show shortcut badge area: existing shortcuts display their combo, empty cards show "+"
3. User clicks the "+" or clicks on an existing shortcut badge to start recording
4. Badge transitions to recording state with pulsing accent border and "按下快捷键..." text
5. User presses a key combination

#### Key Validation Rules

| Rule | Check | On Failure |
|------|-------|------------|
| Must include modifier (D-06) | At least one of Ctrl, Alt, Shift must be in the combo | Show toast: "快捷键必须包含修饰键 (Ctrl/Alt/Shift)" |
| Max 3 keys (D-07) | Total keys in combo must be <= 3 | Show toast: "快捷键最多支持 3 个按键组合" |
| No conflict (KB-04) | `isRegistered(shortcut)` check + scan current project commands for same combo | Show inline conflict state + toast with conflicting command name |
| Single modifier only | Reject combos like "Ctrl" alone (need modifier + non-modifier) | Same as modifier check above |

#### Recording Exit Conditions

| Event | Result |
|-------|--------|
| Valid combo pressed (e.g. Ctrl+G) | Register shortcut, show success flash, transition to display state |
| Conflict detected | Show conflict state (red badge "冲突" + toast), remain in recording for retry |
| Escape pressed | Cancel recording, return to previous state |
| Click outside badge | Cancel recording, return to previous state |
| Edit mode toggled off | Cancel all active recordings, return to non-edit display |

#### Key Format Conversion

```typescript
// Browser KeyboardEvent -> Tauri shortcut string
function keyboardEventToShortcut(e: KeyboardEvent): string | null {
  const parts: string[] = [];
  if (e.ctrlKey || e.metaKey) parts.push("CommandOrControl");
  if (e.altKey) parts.push("Alt");
  if (e.shiftKey) parts.push("Shift");

  // Reject modifier-only combos
  const mainKey = e.key;
  const MODIFIER_KEYS = ["Control", "Alt", "Shift", "Meta"];
  if (MODIFIER_KEYS.includes(mainKey)) return null;

  // Add the main key (uppercase single char, or special key name)
  if (mainKey.length === 1) {
    parts.push(mainKey.toUpperCase());
  } else {
    // Handle special keys: F1-F12, ArrowUp, etc.
    parts.push(mainKey);
  }

  return parts.join("+");
}

// Tauri shortcut string -> Display string
// "CommandOrControl+Shift+R" -> "Ctrl+Shift+R"
function shortcutToDisplay(shortcut: string): string {
  return shortcut.replace("CommandOrControl", "Ctrl");
}
```


### 4. Number Keys 1-9 Coexistence (Claude's Discretion)

#### Decision: Keep number keys 1-9 alongside custom shortcuts

Rationale:
- Number keys are DOM-level shortcuts (useKeyboard.ts), only work when app is focused
- Custom shortcuts are OS-level (tauri-plugin-global-shortcut), work even when app is minimized
- They serve different use cases: number keys for quick in-app use, custom shortcuts for global access
- No conflict possible: number keys have no modifier, custom shortcuts require modifiers (D-06)
- Backward compatibility: existing users expect 1-9 to work

#### Implementation:
- `useKeyboard.ts` remains unchanged
- Number badge display: when a card has a custom shortcut, display the custom shortcut instead of the number
- When a card has NO custom shortcut AND is among the first 9 non-custom/non-editable cards, show the number badge as before

```typescript
// MainArea.tsx — updated shortcut prop logic
```
